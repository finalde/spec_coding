## 使用说明（先读这里）

1. **所有 Prompt 都在三重反引号代码块里**，直接复制整块粘贴到 Seedance 或 Kling。
2. **Prompt 正文语言一律英文**——英文是两个生成器最稳定的输入语言；中文参考翻译仅用于你读懂 Prompt 含义，不要粘贴中文进 Seedance / Kling。
3. **角色 Prompt 与场景 Prompt 的角色描述必须原封不动 paste**——这是跨镜头人物一致性的核心机制，不要改写。
4. **Kling 3.0 的 Negative Prompt 是一个独立输入框**，Kling 界面会专门提供；Seedance 2.0 没有独立负向提示词字段，已在正向 Prompt 末尾的 `no X, stable Y` 句式里写好。
5. **9:16 竖版、15 秒、24 帧**——已写入每条场景 Prompt 末尾，不要改动。
6. **上传前必须开启 YouTube Studio → Details → Altered content → Yes**（2026 年 AI 内容强制披露），描述里再写一遍披露句仍需保留。
7. **一镜头一运镜、一拍子一动词**——多个运镜词叠加或多个动词叠加是 Seedance / Kling 最常见失败触发器。

## 1.7 模型备注与失败风险
- **为何选 Seedance 2.0：** Environment Lock + 95% 花纹保留在"猫—动作—反应"三拍里稳定度最高；Kling 对长镜头静态猫咪会漂移面部。
- **主要失败点：** 爪子按阀门时人手可能意外出现——场景 Prompt 里明写 `a human hand off-frame`，不要让模型把手画在主视角里。
- **花纹漂移缓解：** 角色 Prompt 的 M 纹、白胸毛、左前爪白袜必须在每次生成里逐字原样出现。
- **音频策略：** Seedance 2.0 自带 Foley，espresso 机嘶嘶声自动合成；Kling 也能，但 Negative Prompt 要写 `no background music` 保持纯 Foley。

## 10.7 模型备注与失败风险
- **为何选 Seedance 2.0：** 双角色对话 + 连续环境是 Seedance 强项；Kling 对 "alien + human" 同框时人类脸可能飘。
- **主要失败点：** 脸颊青点多出或消失是最常见失败——Negative Prompt 必写 `extra cheek dots, more than two cheek dots`。
- **对口型建议：** 每人一句、每句 ≤14 词；Kling 使用 dialogue 语法；Seedance 让 ElevenLabs 单独生成 ZARN 声线后对口型。
- **文化安全：** 问题保持"荒诞但无政治不正确"——类似 "why do humans clap / why is it called fast food / why do humans say bless you"——不触碰种族、宗教、政治。

# 附录 A — 拍摄后处理 Checklist（上传前必做）

1. **AI 披露开关：** YouTube Studio → Upload → Details → Altered content → **Yes**（强制）。
2. **水印去除：** 跨平台二次上传时必须抹掉原平台水印，否则 IG / YT 会降权（Socialync 重编码流水线或 CapCut 直接导出）。
3. **首帧 1.5s 钩子测试：** 自己静音看前 1.5 秒，能不能知道发生了什么？不能就重剪。
4. **标题 40 字符测试：** iPhone Shorts feed 实机预览，看截断是否在正确位置（中文字符也算 1 字符）。
5. **前 3 个 hashtag 确认：** 第 1、2、3 号 hashtag 会渲染在标题上方——确保是最精准的三个（`#Shorts` + 一个题材标签 + 一个工具/内容标签）。
6. **尾帧 CTA：** 即使 15 秒也要有"关注 / Subscribe"视觉元素，出现在第 13-15 秒。
