# 情节对白大师（plot & dialogue master）

审查并**直接修改** AI 短剧（`ai_videos/{name}/`）每个 shot 与每集的**情节 + 对白**，标准是「一个普通观众看着会不会觉得假、空、跳、看不懂、没共鸣」。发现不对 / 不完善 → **当场改写**（surgical，不重写整集），并把改动同步到 shot 的 `台词 / 字幕:` 行、`dialogue.md`、`script.md`。

本 skill 是 `agent_refs/project/ai_video.md` §12.4-D + `agent_refs/validation/ai_video.md` move #9 既有「短剧故事+台词大师」契约的**可执行版**——不仅打分，还落地修改。

## 何时用
- 用户说：对白太浅 / 不像正常人说话 / 像喊口号 / 缺前因后果 / 情节有漏洞跳跃 / 金手指(系统)没互动 / "审一下台词""把对白改深""润色""检查这集剧情"。
- 新写 / 重剪某集后的收尾把关。

## 输入
- 目标范围：某 shot、某集（`episodes/epNN/`）、或整个 drama。
- 必读：该集 `script.md` / `dialogue.md` / `shotlist.md` / `shots/shotNN.md`；角色 bible（声口/动机）；`world.md`（设定/系统）。

## 台词审查标准（每行台词逐条过）

| # | 准则 | 不合格信号 | 怎么改 |
|---|---|---|---|
| D1 | **说人话** | 像标语 / 书面口号 / 文绉绉硬邦邦 / 没人会这么说话 | 改成口语、有语气、有停顿、有称呼 |
| D2 | **因果交代** | 只下结论不讲缘由（"裴家不养废物"→为什么？） | 补前因后果："你丹田碎了→修不了武→裴家以武立命→容不下废人" |
| D3 | **信息量** | 一句废话、不推进剧情也不揭示动机/stake | 让每句话推进情节、揭示人物关系/动机/利害，或埋钩 |
| D4 | **角色声口** | 谁说都一样、不贴身份性格（bible §角色定位/气质） | 按该角色身份/性格/处境调措辞（嚣张少主≠威严家主≠软弱母亲） |
| D5 | **情绪真实** | 平、假、用力过猛、不符合此刻处境 | 贴合此刻情绪与关系；恨里有由、护里有怯 |
| D6 | **长度合宜** | 一律砍到≤15字导致干瘪；或啰嗦拖沓 | 该交代因果/情绪的关键台词可长(自然口语)，废话砍掉 |

## 情节审查标准（每个 beat / 每集过）

| # | 准则 | 不合格信号 | 怎么改 |
|---|---|---|---|
| P1 | **情节链完整** | beat 之间跳、缺因果、观众看不懂为什么 | 补衔接/动机镜或台词，让每步是上一步的合理结果 |
| P2 | **动机可信** | 角色行为没理由 / 为反派而反派 | 给动机（利益/情感/性格）；反派也要有自己的逻辑 |
| P3 | **爽点铺垫+落地** | 爽点突兀没铺垫，或喊了没落地 | 前面埋因(委屈/不甘/前情)，爽点处用"行动→反应"落实 |
| P4 | **金手指/系统互动** | 系统只单方面弹面板、主角毫无反应 | **主角必须与系统有来回**：发现→质疑/调侃→抉择→确认；系统有"人设"(冷/萌/古板)，一问一答 |
| P5 | **观众共情** | 普通观众无感、不揪心不解气 | 强化共情锚（被至亲背叛/救人反被害/绝境不屈）与解气点 |
| P6 | **设定自洽** | 与 world.md / 既定 canon 矛盾 | 对齐 canon；冲突则按 spec/world 修 |

## 工作流（review → 直接改）
1. 读范围内 `script.md`+`dialogue.md`+`shots/*.md` + 角色 bible + world.md。
2. 逐行台词过 D1–D6、逐 beat 过 P1–P6，列出不合格项（cite shot/行/准则）。
3. **直接改写**不合格台词与情节（surgical：只改该行/该 beat，不重写整集；保留 shot 结构、锁定串、时长）。改写要：说人话 + 给因果 + 贴声口 + 有信息量。
4. **金手指/系统**：若该集有系统而无主角↔系统互动，**补一段来回对话**（主角发现/质疑/调侃 + 系统回应），写进相应 shot 的 `台词 / 字幕:` 与 `dialogue.md`/`script.md`。
5. **三处同步**：shot 的 `台词 / 字幕:` 行、`dialogue.md`、`script.md` 改后保持一致。
6. 复验：shot 数/时长/锁定串/零hex 不变（台词层 surgical）；台词读起来像真人对话、有因果、有互动。

## 硬约束（沿用项目规则）
- 台词只作**字幕**（v1 visual-only，不下发 TTS）；OS/画外/系统音须带 `· 在画人物口型:` 闭口指令。
- 系统提示字幕用鎏金描边区分人物对白。
- 不引入 world.md 未有的设定；不剧透后续 arc 专名。
- 改写后台词仍是中文、零 hex；不动比例（用户自设）。

## 输出
- 一份 inline patch 清单（shot / 原行 / 准则 / 改写后）+ 已落地的文件改动 + 一句话复验结论。
