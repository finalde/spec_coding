# 动作 / 表演大师（action master）

审查并**直接修改** `ai_videos/{name}/` 每个 shot 的**动作与表演**：人物在每个 beat 具体做什么、配角对正在发生的事有没有真实反应、动作有没有把这个镜该落的钩子落地。失败模式是"干站着空对镜头""配角被偷袭还像木头人""权势者冷酷宣判却端坐正脸絮叨"。发现不合理 → 当场 surgical 改 `动作:`（必要时 `情节:` 的物理描写），不改台词措辞、不改运镜。

本 skill 承接原 `ai_videos__台词大师` 的 P7（表演/走位合常理）+ `ai_video.md` 12.4-E（小说级细节密度）+ 12.4-D S1（钩落）/S2（情节链非装饰）。由 `ai_videos__审查总编排` 在单镜层调用（运镜之后、光线之前）。

## 何时用
- 用户说"动作/表演不对""木头人/干站着""配角没反应""动作没落钩/太空/没细节/没微表情"。
- `ai_videos__审查总编排` 单镜层第 4 道工序。

## 输入
- 范围：某 shot / 某集 / 整剧。
- 必读：被审 shot 的 `动作:`/`情节:`/`节奏:` + 该镜在 shotlist/arc 上的钩标记（黄金钩/反转/cliff）+ 角色 bible（性格决定反应方式）。

## 审查标准（每 beat / 每在画人物过）

| # | 准则 | 不合格信号 | 怎么改 |
|---|---|---|---|
| A1 | **动作具体 + 配时间轴** | `动作:` 笼统（"主角很生气"）、无分秒 beat、铺不满时长 | 拆成带时间窗的具体物理动作（`0-2s 双目猛睁、瞳孔骤缩；2-5s 闷哼撑住青砖…`），beats 铺满 shot 时长。⚠ 动作 beats 总时长 vs shot 时长的**合理性裁决**（演不演得开、要不要加时长/拆镜）归 `ai_videos__时长节奏`（PA3）；本项只管动作本身具体、有时间窗。 |
| A2 | **钩落（S1）** | 该镜在 arc 上标了钩（黄金钩/反转/cliff），动作却只是站着，钩没视觉落地 | 重排动作让钩在 `镜头+动作+台词` 里**可见地发生**在其声数内 |
| A3 | **非装饰 beat（S2）** | 整镜是可删的装饰动作（30s 主角站着），不是情节链必需一步 | 把动作绑到一个具体情节转折；删纯装饰或并镜 |
| A4 | **配角反应合理（P7）** | 在画配角对眼前事件毫无反应（被偷袭还呆站、shot11 教训）；权势者姿态不贴情绪（冷酷下逐令却端坐正对台下絮叨、shot12 教训） | 每个在画人物按处境给**合理反应**（被攻击→惊呼/躲闪/踉跄/护人；受辱→变色/僵住）；走位体态贴权力关系（凉薄宣判可背对/负手/头也不回） |
| A5 | **四层密度（12.4-E）** | 长镜（≥8s）只有 macro 动作，缺微表情/身体朝向/氛围层 | 每 beat 补四层：macro 动作 + 微表情 + 身体朝向&视线目标 + 氛围（光影/环境随动作）；短镜（<8s）至少 macro+微表情 |
| A6 | **多角色反应差异化** | 群像反应镜像复述（"A 侧身朝沧冥/B 侧身朝沧冥/C 侧身朝沧冥…"） | 每个角色承载差异信息（不同 gaze target/不同微表情/不同物理姿态），不是同句换名字 |
| A7 | **表演拟人化·情绪层次（短剧优化铁律）** | 动作机械完美、走位规整像机器人；一个情绪只配一个单调动作（回望只「蹙眉摇头」、离别无怅然、领大礼只「肩背一颤」）；缺微表情、松弛感 | 去机械完美走位、动作松弛真实有人性；关键情绪给**层次/转折**——回望先一丝极淡怅然→转瞬被冰冷决绝覆盖；步伐由虚浮→逐步沉稳；嘲讽由轻佻→转狠戾；生疑先诧异→细辨→无果敛神。让角色像活人、不像逆袭机器（2026-06-17 EP2 全镜优化教训） |

## 工作流
1. 读 shot 的 `动作:`/`情节:`/`节奏:` + 钩标记 + 角色性格。
2. 逐 beat / 逐在画人物过 A1–A6，列不合格项。
3. **当场 surgical 改** `动作:`（必要时 `情节:` 物理层）：补具体动作+时间窗、落钩、给配角反应、加四层密度、差异化群像。beats 须铺满时长。
4. 复验：没有干站镜、钩都落地、配角都有反应、长镜细节够、群像不雷同。

## 硬约束
- 只动**动作/表演/物理 beat**，不改台词措辞（`台词大师`）、不改朝向硬约束（`站位朝向`）、不改运镜（`运镜`）、不改剧情因果（`剧情连贯`）。
- **眼睛不写发光特效**（瞳泛金/瞳孔发光）——能量入体改"眉心一缕微光内敛"，眼变化靠表演；发光相关归 `光线色调` 复核。
- 改时长须与 `台词大师` 的台词节奏对齐（动作 beats 铺满新时长，别只改时长不改 beats）。
- 审计写 `.audit/.../events.jsonl` 的 `validation.issue.raised`（level=action）。

## 输出
- inline patch 清单 + 已落地改动 + 一句话复验结论。
