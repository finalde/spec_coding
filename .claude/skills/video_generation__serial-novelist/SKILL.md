# 连载短篇小说 · Serial Novelist — 单集 900–1200 字爽文 + 6–8 段分镜

生产 **单集** 电视化分镜级短剧原稿。写完的这一集，既能作为小说独立读，也能 1:1 喂给 `video_generation__storyboard-master` 出分镜，不用任何回问。

## Modes

- **新系列启动模式 (bootstrap)** —— 用户只给一句题材 / 一句开局。产出：`series/{name}/README.md` + `status.md` + 第 1 集 `ep_001.md` + 次 3–5 集 `outline/next_5.md` + 整系列 `outline/arc_overview.md`（只写 50–80 集的一句话概要，不展开）。bootstrap 完成后必须交给 `video_generation__preproduction` 锁四圣经。
- **续集推进模式 (rolling)** —— 从 `series/{name}/outline/next_5.md` 和上一集 `ep_{N-1}.md` 继续往下写 1–3 集。每次最多 3 集。写完后更新 `next_5.md` 与 `arc_overview.md`。

## Workflow

### Step 1 — 读入上下文

必读：

- `series/{name}/status.md`（确认 `bibles` 状态、目前到哪一集）
- `series/{name}/bibles/*`（rolling 模式强制，bootstrap 模式跳过）
- `series/{name}/bibles/anchor_registry.md`（rolling 模式强制）
- `series/{name}/outline/arc_overview.md` + `outline/next_5.md`（rolling 模式）
- `series/{name}/episodes/ep_{N-1}.md`（rolling 模式，用于前后集连贯钩）

### Step 2 — 题材 / 视角硬校验

- 题材：**重生 / 穿越 / 系统 + 古风 / 仙侠 / 玄幻**（Q1bc）。其他题材一律拒绝并提示用户走另开系列。
- 视角：**大男主**（Q2b 战神归来 / 扮猪吃虎 / 赘婿翻身 中择一路）。双主视角 / 女频 / 群像 → 拒绝。
- 角色上限：**2–3 个**（Q8a：男主 + 1 女配 + 1 师父 / 反派 / 系统）。同集超员 → 拒绝。
- 弧线：**纯爽文**（Q6d）—— 不允许真正失败 / 长线受辱 / 情绪崩溃 / 低谷桥段。允许「装」一集，不允许「真输」。

### Step 3 — 滚动式大纲（Q7c）

- **只规划未来 3–5 集**。拒绝一次性写 50–80 集细纲。
- bootstrap：`arc_overview.md` 列 50–80 集 **一句话** 概要（空也可），`next_5.md` 细化最近 3–5 集（每集：一句钩、一次打脸、一个结尾悬念）。
- rolling：读 `next_5.md`，写 1–3 集，然后把 `next_5.md` 向前滚动（第一行丢掉，尾部补新的一行）。

### Step 4 — 单集正文

逐集产出的 `episodes/ep_{NNN:03d}.md` 必须满足：

- **字数 900–1200 汉字**（Q4c，对应 ~2 min TTS）。超出 → 压缩；不足 → 补动作 / 旁白密度，而不是水字。
- **每 250 字一次爽点 / 反转**，每集至少 2 次、推荐 3 次。
- **前 120–180 字** 完成开场钩（3 秒内抛冲突）。
- **末 100–150 字** 留 **H 型或 J 型钩子**（见 §Hook Templates）。
- **打脸三段式** 至少嵌入一次：`踩（反派羞辱） → 亮（主角亮底牌） → 碾（权威崩塌 / 反派自灭）`。
- **身份反差** 是主线（外卖员 / 赘婿 / 废物 ↔ 战神 / 宗主 / 仙尊 / 系统宿主）。
- **台词**：单句 ≤10 汉字；不堆长独白；不做心理戏长段。
- **合规红线**：无血腥断肢描写；反派死法走 `自灭 / 反噬 / 天谴 / 更高势力斩杀`，不由主角直接血腥手刃；不出现 `算命 / 风水 / 招魂` 作为现实可行的指引。

### Step 5 — 6–8 段分镜拆解（Q10c 四栏）

把正文等切成 **6–8 段**（均值 7），每段 **≤15 秒** TTS 时长（即 60–120 字 VO 含停顿），四栏逐行填。**全集总时长 90–120 s**（6 段 × 14.4 s ≈ 86 s → 8 × 14.4 s ≈ 115 s，含 Seedance 首末 0.3 s 修边容差）：<90 s → 加段或加 BGM 尾奏；>120 s → 压缩最长段。

| # | 画面 | 对白 | 旁白 | 情绪节拍 |
|---|------|------|------|---------|
| 1 | {景别 + 运镜 + `{{character:...}}` + `{{scene:...}}` + 动作} | `【角色ID】"≤10 字"` | 可为空或 1–2 短句 | 踩 / 亮 / 碾 / 钩 / 垫 |
| … |

每段还要给出：

- **景别**：`大特写 / 特写 / 近景 / 中近景 / 全景 / 远景` 之一
- **运镜**：`推近 / 拉远 / 跟拍 / 平移 / 环绕` 之一
- **道具清单**：本段画面内必须出现的道具（从 `scene_bible` 复用片段里延伸出的细节）
- **TTS 音素提示**：本段 VO 中的多音字 / 生僻字 pinyin 标注（例：`璟=jǐng` `砚=yàn`）
- **字幕安全文本**：本段字幕切分，单行 ≤15 汉字，用 `|` 分隔断行（例：`十年前我埋的剑|该出鞘了`）

### Step 6 — 15 秒快照钩 + 前后连贯钩

- 在相邻两段之间写一条 **15 秒快照提示**（段 N 末帧 = 段 N+1 起始帧）：景别 / 人物姿态 / 表情 / 道具状态 / 服装状态 / 背景锚定元素 / 光线色温。
- 本集首段起始帧 = 上一集末帧；本集末段末帧 = 为下一集留帧（写明身份未揭 / 境界未破 / 强敌压境 / 美人生死 这四类之一悬念）。

### Step 7 — 内联 bible 引用 + 复用片段

- 所有人物 / 场景描写只能通过 `{{character:name}}` / `{{character:name.view}}` / `{{scene:id}}` / `{{scene:id.view}}` / `{{style:anchor}}` 锚点引用。禁止自由描写五官 / 服装 / 场景色温（这些是 `preproduction` 的权限）。
- 每个锚点首次在本集出现时，必须 **逐字粘贴** 对应的 `复用片段`（便于本集 md 作为独立单元喂 Seedance 时不用回查）。
- 引用未在 `anchor_registry.md` 登记的锚点 → 中止并指回 `preproduction`。

### Step 8 — 结尾钩 + 更新 outline

- 把本集钩子从 §Hook Templates 中选一条编号记录（例：`末钩: H1 - 下集预告卡点`）。
- 更新 `outline/arc_overview.md`：给本集补一句话概要。
- 更新 `outline/next_5.md`：头部删掉已写的那行，尾部按滚动规则补一行新计划。
- 更新 `status.md`：`ep_{NNN}: written`。

## Per-Episode Output Contract

`episodes/ep_{NNN:03d}.md` 必须按顺序含以下 H2 章节（Q9d 全集 + 复用）：

1. `## 集号 + 标题`
2. `## 登场角色 + 场景锚点`（列出本集使用的 anchor + 逐字复用片段）
3. `## 正文`（900–1200 字散文）
4. `## 分镜四栏`（6–8 行表格 + 景别 / 运镜 / 道具清单）
5. `## TTS 音素提示`（多音字全表）
6. `## 字幕安全文本`（按段编号，`|` 分行）
7. `## 15 秒快照钩`（相邻段 N → N+1 的末帧描述）
8. `## 前后集连贯钩`（开场帧 = 上集末帧；末段帧 = 下集悬念）
9. `## 结尾钩`（标记 Hook Template 编号 + 台词）
10. `## 本集元数据`（总字数 / 段数 / 总时长估算 / 爽点数）

## Hook Templates

**编号即合约**（结尾钩必须从 H/J 类里选 1 条；打脸场景从 B/I 里选；每 5 集至少 1 次 C/D/E 类金手指 / 境界钩）。

- **A · 身份反差钩** —— `"他只是个{低位}，谁能想到，他曾是{高位}。"`
- **B · 反派嘲讽即将翻车钩** —— `"就凭你？也配{动作}？" / "她不知道，她面前这个{弱态}的男人，三息之前刚{惊天动作}。"`
- **C · 系统触发钩** —— `"叮——宿主{触发}，检测到{事件}，奖励{物}。"`
- **D · 重生预知钩** —— `"上一世，他在这里{惨剧}。这一世，他要让所有人{反向}。"`
- **E · 境界突破钩** —— `"{动作}的那一刻，他体内{灵气/丹田}轰然{动}，{旧境界}破碎，{新境界}降临。"`
- **F · 强者来访钩** —— `"一道身影自{方位}而来，整座{地点}齐刷刷{动作}——除了他。"`
- **G · 退婚 / 悔婚钩** —— `"{反派}拿出{信物}，冷笑：'从今日起，你不再是我{关系}。'他只是笑了笑，从怀里慢慢掏出了{反转物件}。"`
- **H · 下集预告卡点钩**（**每集结尾必须强制用**） —— `"就在此时，{意料外势力}突然{动作}。所有人脸色一变——{悬念句}。"`
- **I · 饭局 / 宴会突发钩** —— `"酒席上，{反派}举杯讥讽：'{羞辱台词}。'满座哄笑。唯独{关键人物}脸色骤变——他认出了{主角隐藏标志}。"`
- **J · 反派自灭钩**（**替代血腥**） —— `"{反派}狂笑未止，忽然{生理反应}，喉中溢出一缕{超自然现象}——'你……你不是凡人！'话音未落，{倒地/化灰/魂飞魄散}。"`

付费卡点（**第 10 / 30 / 60 集结尾必须用硬悬念**）：身份未揭 / 境界将破 / 强敌压境 / 美人生死 四选一。

## Inputs

- **bootstrap**：用户口头 pitch（一句题材 + 一句开局）。
- **rolling**：`series/{name}/status.md` + `bibles/*` + `outline/*` + `episodes/ep_{N-1}.md`。

## Outputs

- `series/{name}/episodes/ep_{NNN:03d}.md`（本次新写的 1–3 集）
- 更新 `series/{name}/outline/arc_overview.md`
- 更新 `series/{name}/outline/next_5.md`
- 更新 `series/{name}/status.md`
- **bootstrap 附加**：`series/{name}/README.md` + `status.md` 初始化（但四圣经由 `preproduction` 写，不在此处）

## 输出规范

- **全部中文**；只有锚点 token（`{{...}}`）、hex 色值、voice_id 型号、`4K` / `DoF` 保留英文。
- **字数 900–1200 / 集**，硬边界；单段分镜 TTS ≤15 s（60–120 字 VO 含停顿）。
- **单句台词 ≤10 汉字**；不堆长独白。
- **每 250 字一次爽点 / 反转**；每集 ≥2 次打脸或身份反差；每集 ≥1 次 H/J 类结尾钩。
- **不写心理长段**；情绪靠动作 + 对白 + 反应镜头承载。
- **2–3 个同集在场角色**；超员拒绝。
- **禁用空泛形容词**（沿用 `topic-scout`：`好看 / 唯美 / 漂亮 / 大气 / 震撼 / 炸裂` 全禁）。
- **锚点必须登记**；未在 `anchor_registry.md` 的锚点 → 中止。
- **复用片段必须逐字粘贴**；不允许同义改写。
- **合规红线**：无血腥断肢 / 无真实宗教仪式指引 / 重生原因走系统 / 雷劈 / 天道而不走 `阎王送回`；反派死法走自灭 / 反噬 / 天谴 / 上位势力。
- **不一次规划超过 5 集**；用户强要求 → 仍然只写 3 集并提示滚动规则。

## 边界情况

- 用户给了非锁定题材（霸总 / 甜宠 / 都市正剧 / 女频）→ 拒绝并建议另起系列。
- 用户想写第 4 个同场角色 → 拒绝并给 2–3 套合并方案。
- 用户想加悲剧 / 低谷 / 主角阶段性失败 → 拒绝；提示「装输不允许真输」。
- 上一集钩子与本集开场不匹配 → 优先改本集开场，不改上一集；如必须改上一集，走 `edit_prev_episode` 并写 CHANGELOG。
- 锚点引用了 `anchor_registry.md` 还没有的条目 → 中止并列出缺失锚点，指回 `preproduction`。
- 用户要求一次 4+ 集 → 只写 3 集并说明滚动约束。
