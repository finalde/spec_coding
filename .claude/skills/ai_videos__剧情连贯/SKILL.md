# 整集剧情连贯大师（continuity master）

审查并**直接修改** `ai_videos/{name}/` **一集之内**的剧情逻辑与镜间衔接：情节链完不完整、动机可不可信、爽点有没有铺垫并落地、主角与系统有没有来回互动、相邻镜接不接得上、有没有跨镜重复的台词/视觉、本集内专名出现时序对不对。发现不合理 → 当场 surgical 改（补衔接镜/动机台词、改重复 beat、删冗余），三处同步（shot/dialogue.md/script.md）。**不重写整集**。

本 skill 承接原 `ai_videos__台词大师` 的 P1–P6 + CLAUDE.md「改动剧本/台词后默认连贯性 check」相邻镜衔接 + `ai_video.md` 12.4-D S2/S3/S5。**跨集**的查重与递进归 `ai_videos__全剧序列`。由 `ai_videos__审查总编排` 在整集层调用（全镜单镜层完成之后）。

## 何时用
- 用户说"剧情/连贯性不对""情节漏洞/跳跃""动机不可信""爽点没落地""系统没互动""相邻镜不顺/重复""这集穿帮"。
- 写完/重剪某集后的整集把关；`ai_videos__审查总编排` 整集层工序。

## 输入
- 范围：某集（`episodes/epNN/`）。
- 必读：该集 `script.md` + `dialogue.md` + `shotlist.md` + 全部 `shots/shotNN.md`；角色 bible（动机/性格）；`world.md`（设定/系统/canon）；`arc_outline.md`（钩与爽点布局）。

## 情节审查标准（每 beat / 整集过）

| # | 准则 | 不合格信号 | 怎么改 |
|---|---|---|---|
| P1 | **情节链完整** | beat 之间跳、缺因果、观众看不懂为什么 | 补衔接/动机镜或台词，让每步是上一步的合理结果 |
| P2 | **动机可信** | 角色行为没理由 / 为反派而反派 | 给动机（利益/情感/性格）；反派也要有自己的逻辑 |
| P3 | **爽点铺垫+落地** | 爽点突兀没铺垫，或喊了没落地 | 前面埋因（委屈/不甘/前情），爽点处用"行动→反应"落实 |
| P4 | **金手指/系统互动** | 系统只单方面弹面板、主角毫无反应 | **主角必须与系统有来回**：发现→质疑/调侃→抉择→确认；系统有人设（冷/萌/古板），一问一答 |
| P5 | **观众共情** | 普通观众无感、不揪心不解气 | 强化共情锚（被至亲背叛/救人反被害/绝境不屈）与解气点 |
| P6 | **设定自洽** | 与 world.md / 既定 canon 矛盾 | 对齐 canon；冲突按 spec/world 修 |

## 镜间衔接审查（相邻镜 + 整集序列）

| # | 准则 | 不合格信号 | 怎么改 |
|---|---|---|---|
| N1 | **相邻镜衔接** | 本镜与前/后镜的动作/站位/朝向/情绪/台词承接突兀、跳脱，或同一 beat（如气场反转）相邻镜重演 | surgical 改相邻镜或本镜，使承接顺滑、关键转折只发生一次（不重演，只承接"已发生"状态） |
| N2 | **台词跨镜不重复（S5）** | 两个不同镜/角色用近乎相同措辞下威胁/宣告（"今日便是你的劫数"两处） | 差异化措辞/比喻/节奏；同一"名场面"宣告只留一处 |
| N3 | **视觉/beat 跨镜不冗余** | 相邻镜情节 beat 重复（同一动作演两遍）、或整集有可删的重复段 | 合并或推进；让每镜是情节链不可跳的一步 |
| N4 | **本集内专名时序（S3）** | 台词提前出现要后续 arc 才登场的人/物专名（ep01 提到 ep14 才出的"璃月"） | 换成在场可指代（"弟子"/场景级指代），专名等其入场再用 |

## 工作流
1. 读该集 `script.md`+`dialogue.md`+全 `shots/*.md` + bible + world.md + arc_outline.md。
2. 逐 beat 过 P1–P6，把全镜排成序列过 N1–N4，列不合格项（cite shot/beat/准则）。
3. **当场 surgical 改**：补衔接/动机/系统互动、改重复 beat 与重复台词、删冗余、换提前专名。改后**顺着往后改受影响的镜**（空间/状态单调连续，参 CLAUDE.md 走位轨迹核对）。
4. **三处同步**：shot 的 `情节:`/`台词:`、`dialogue.md`、`script.md`。
5. 复验：情节链通、动机可信、爽点落地、系统有互动、相邻镜顺、无跨镜重复、无提前专名。

## 硬约束
- 改剧情/因果/衔接/系统互动；台词逐行措辞质量归 `台词大师`、朝向归 `站位朝向`、运镜归 `运镜`、动作 beat 归 `动作表演`——本 skill 改的是"剧情逻辑与镜间关系"，落到这些字段时与对应 skill 不冲突即可（冲突让位于硬约束类的 blocking）。
- **跨集**问题（开场重复上一集结尾、签名台词跨集复用、爽点跨集递进）**不在本 skill**，交 `ai_videos__全剧序列`。
- 静音/默剧不算修好剧情——背后情节/走位不合理要顺着往后改（CLAUDE.md 全剧序列 review 第 5 点）。
- 不引入 world.md 未有的设定；不剧透后续 arc 专名。
- 审计写 `.audit/.../events.jsonl` 的 `validation.issue.raised`（level=continuity）。

## 输出
- inline patch 清单 + 已落地改动 + 一句话复验结论。
