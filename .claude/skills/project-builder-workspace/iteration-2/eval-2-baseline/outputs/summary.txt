Project: tools/slack_notifier
Purpose: Sends Slack notifications when CI pipelines finish.

Files created
-------------
tools/slack_notifier/main.py
  - CLI entry point (~35 lines). Parses --pipeline, --status, --branch,
    --commit, --url, --duration, --triggered-by flags.
  - Reads SLACK_BOT_TOKEN and SLACK_CHANNEL from environment.
  - Constructs a CIPipelineEvent, instantiates SlackNotifier, calls notify().

tools/slack_notifier/libs/__init__.py
  - Re-exports SlackNotifier and CIPipelineEvent.

tools/slack_notifier/libs/notifier.py
  - CIPipelineEvent: frozen dataclass with full validation in __post_init__.
    Fields: pipeline_name, status (Literal), branch, commit_sha, build_url,
    duration_seconds (int | None), triggered_by (str | None).
  - SlackNotifier: owns WebClient + channel. Method notify(event) posts a
    rich Slack attachment (color-coded, with fields for branch, commit,
    triggered_by, duration). Returns message ts. Raises SlackApiError on
    API failure.
  - _format_duration: pure static helper, converts seconds to "Xm Ys" string.

Design decisions
----------------
- Strong typing throughout (type hints on all params, returns, locals).
- PipelineStatus defined as Literal["success","failure","cancelled"] for
  exhaustiveness.
- CIPipelineEvent uses @dataclass(frozen=True); validation raises ValueError
  for invalid input so callers get clear errors at construction time.
- SlackNotifier is object-oriented; state (client, channel) owned by class.
- main.py is thin (~35 lines): only arg parsing + env reading + one call to libs.
- Color and emoji maps kept as class-level dicts for easy extension.

Exit codes
----------
0  - Notification sent.
1  - Missing env vars or Slack API error.
