  requirements.txt      Empty (no external dependencies; stdlib only).

    fetcher.py          Contains two classes:
                          - FetcherConfig  (@dataclass frozen=True)
                              Holds base_url, output_path, timeout_seconds,
                              headers. Validates inputs in __post_init__.
                          - Fetcher
                              fetch()           -> dict   (GET + parse JSON)
                              fetch_and_save()  -> Path   (fetch + write file)
                              _build_url()      -> str    (internal helper)

requirements.txt        Mock root requirements.txt with:
                          -r projects/data_fetcher/requirements.txt

### Design decisions

1. No third-party dependencies: urllib (stdlib) handles HTTP; json + pathlib
   handle serialisation and file I/O.
2. Separation of concerns: FetcherConfig owns validation, Fetcher owns
   behaviour; main.py owns CLI only.
3. Strong typing throughout (Python 3.10+ union syntax, all parameters and
   return values annotated).
4. OOP-first: domain concepts modelled as classes, not free functions.
5. Thin entry point: main.py is 15 lines and contains zero business logic.
