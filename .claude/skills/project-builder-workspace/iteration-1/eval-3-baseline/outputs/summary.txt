== What was built ==

A server log analyzer that reads a log file line-by-line, matches known log
formats with a single compiled regex, and produces a structured summary report.

== Files created ==

projects/log_analyzer/
  main.py               CLI entry point (~15 lines); delegates entirely to libs/.
  requirements.txt      No third-party deps (stdlib only).
  README.md             Documents usage, supported formats, classes, and exit codes.
  libs/__init__.py      Empty package marker.
  libs/analyzer.py      All application logic.

== Key design decisions ==

- Analyzer class owns state (_log_path, _entries, _unparsed) and validates input
  in __init__ (raises FileNotFoundError / TypeError / ValueError for bad input).
- parse() returns list[LogEntry] and caches results; summarize() calls parse()
  automatically if not yet called — caller can use either method independently.
- LogEntry and SummaryReport are frozen dataclasses (immutable value objects).
- SummaryReport.__str__() renders the human-readable report; no print logic in
  the library layer.
- Full type hints throughout; union syntax (X | None) for Python 3.10+.
- Zero third-party dependencies — only stdlib (re, collections, pathlib, dataclasses).
