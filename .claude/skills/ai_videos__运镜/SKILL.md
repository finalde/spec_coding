# 运镜大师（camera master）

审查并**直接修改** `ai_videos/{name}/` 的**镜头语言**：单镜的景别/角度/镜头/运镜对不对，相邻镜之间顺不顺。单镜拍对还不够——**相邻镜的机位/视线/景别要顺承**，否则观众错乱。发现不合理 → 当场 surgical 改（主要改 `镜头:`，必要时 `走位:` 机位关系），不动剧情台词。

本 skill 承接原 `ai_videos__台词大师` 的 M 系列准则（运镜连贯）+ `ai_video.md` 12.4-D S4（视觉密度）。由 `ai_videos__审查总编排` 在单镜层调用（站位之后、动作之前）。

## 何时用
- 用户说"运镜不对/跳/乱/越轴""景别没节奏""反打视线对不上""推拉打架""相邻镜重复或不连贯"。
- `ai_videos__审查总编排` 单镜层第 3 道工序。

## 输入
- 范围：某 shot / 某集 / 整剧（连贯性须按场景把相邻镜排序）。
- 必读：被审 shot 的 `镜头:`/`走位:`/`动作:`、同场景前后镜、`scenes/{name}.md`（机位/方位）。

## 单镜审查（每镜过）
- **景别/角度/镜头匹配 beat**：大特写/特写/近/中/全是否配得上这个戏剧点（反应切特写、群像用中全）；角度（俯/仰/平）贴权力关系；镜头焦段（广角/标准/长焦）合理。
- **运镜有动机**：推/拉/摇/移/跟是否由情绪或动作驱动，不是无理由乱动；静止镜也要明确"固定机位"。
- **影视化动态运镜（短剧优化铁律·默认优先）**：摒弃**死板固定机位**与**呆板对称构图**；竖屏聚焦人脸情绪与关键动作，善用微推拉 / 虚实景深切换 / 情绪特写 / 氛围运镜（远景随人物渐行**极缓抬升**强化城楼压迫感、跟移**贴步伐律动**小幅起伏、**上下/双区分镜**拉对峙张力、台词收尾轻推门外天光做冷暖对冲）。纯静止锁机位只在「刻意凝滞 / 留白」时用、并写明动机（2026-06-17 EP2 全镜优化教训）。
- **比例 9:16**（无项目级 divergence note）。

## 跨镜连贯审查（同场景相邻镜逐对过）

| # | 准则 | 不合格信号 | 怎么改 |
|---|---|---|---|
| M1 | **轴线一致（180°线）** | 同一对话/对峙越轴致人物左右互换（上镜 A 在左、下镜 A 跳右） | 锁定一条人物连线、所有机位待同侧；对峙类全程保持"主角在下/前景、对方在上/高座"的同一相对关系 |
| M2 | **视线方向匹配** | A 看画右、反打 B 也看画右（应相对），眼神对不上 | 反打双方视线**相对**（A 望右→B 望左），高低差一致（被仰拍方下镜仍被仰拍） |
| M3 | **景别有节奏、不硬跳** | 大全景直切大特写、或连续同景别无变化 | 相邻镜景别递进或有动机（远→中→近 / 反应切特写）；跳切处用动作或视线引导 |
| M4 | **运动方向顺承** | 上镜向右移、下镜无理由向左甩；推/拉来回打架 | 连贯段落运动方向顺承；转场用动作、视线或推拉收尾自然衔接 |
| M5 | **站位/进出画一致** | 同一人同场景换镜就换边（沈婉上镜在东、下镜在西）；上镜从左出、下镜从左入 | 同场景同一人**固定一侧**（含其 bg plate 朝向）；出画侧 = 下镜入画对侧 |
| M6 | **同角色全程唯一** | 多次具名+多位置短语致模型渲染出两个同人 | 配合 `站位朝向` C5：每镜同角色只具名一次、其余用称谓，必要时写"全画面共 N 人各一名" |
| M7 | **相邻镜视觉不重复（S4）** | 相邻两镜只差"换个角度"、无剧情/视觉增量（shot04 & shot05 都=主角不同角度站着） | 合并或差异化：给每镜一个视觉上明确不同的 beat（景别/角度/动作/构图至少一项有实质差异） |
| M8 | **跨镜首帧承接判定（continuity-frame，ai_video.md 2026-06-21）** | 强关联的连续镜被当硬切（接缝跳脱、不自然）；或本就该切镜头的对被硬挂了承接帧（与切镜打架） | 逐对判**承接 vs 硬切**：满足①同场景/同bg ②机位连续或渐变(同轴推/拉/移·match cut、非越轴/正反打/景别跳切) ③动作或情绪不间断 → **承接**；任一不满足 → **硬切（默认）**。写死该镜 `## Shot context` 的 `衔接:` 字段（`承接 shot{NN} 末帧（首帧＝上一镜末帧）` / `硬切（独立首帧）`）。承接镜须确认其 `动作:` 首拍 + `走位:` 起始与上一镜末拍续接（不跳变）；帧源＝截上一镜实际成片末帧、禁另生成静帧。落 `参考:`/`Reference uploads` 三处由 `格式契约` K26 机械校验 |

落点：检查每场景 shot 序列的 `镜头:`/`走位:`；冲突改后镜（或统一机位/朝向）。回忆段滤镜一致性归 `光线色调`。

## 工作流
1. 把目标场景的连贯镜按序排好；逐镜过单镜项、逐对过 M1–M7。
2. 列不合格项（cite shot/准则）。
3. **当场 surgical 改** `镜头:`（必要时 `走位:` 机位关系）；优先改后镜以顺承前镜。
4. 复验：轴线不乱、视线相对、景别有节奏、运动顺承、站位固定、无相邻镜雷同；**逐对判 M8 首帧承接、写死每镜 `衔接:` 字段**。

## 硬约束
- 只动**镜头/机位/景别/运镜/构图**，不改台词（`台词大师`）、不改人物朝向硬约束（`站位朝向`，冲突时让位于它）、不改剧情（`剧情连贯`）。
- 比例由用户自设；默认 9:16，改则记 divergence note。
- 审计写 `.audit/.../events.jsonl` 的 `validation.issue.raised`（level=camera）。

## 输出
- inline patch 清单 + 已落地改动 + 一句话复验结论。
