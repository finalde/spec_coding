## 2. 内建提问清单（AskUserQuestion，多选，标默认）

> 世界观 + 人设是**重中之重**——人脸/服化/声口在这一步锁死，后面才不漂。**多问、多角度**：先问世界观大盘，再**逐角色**问锁定字段。用户可随时说"这段自动"跳过。每题用 AskUserQuestion 多选 UI，默认项标 ✅。

**A. 世界观大盘**
1. **时代 / 地域**：✅ 古风（架空王朝 / 仙侠门派）｜ 民国 ｜ 现代都市 ｜ 末世 ｜ 其它。
2. **力量体系 + 特效尺度**：✅ **藏锋·无外放**（无金光 / 无瞳光 / 无外放威能；力量靠体内暖流 + 微表情 + 表演体现）｜ 低度外放（仅高潮镜可见微光）｜ 高玄幻外放（慎选，AI 易乱加特效）。
3. **势力划分**：几方对立？（如 朝堂 / 门派 / 江湖 / 暗线组织）——列名 + 各自立场 + 与主角关系。
4. **角色清单**：主角（1）｜ 反派（1–2）｜ 关键配角（1–3）｜ 暗线 / 剪影角色（0–2）——逐个确认是否立档。

**B. 逐角色锁定（每个具名角色都问一轮）**
对清单里每个角色，依次问（默认值取自 concept + 赛道）：
5. **面部 = 用户选角，不再问五官（2026-06-18 选角供脸）**：用户有角色库，脸经 `casting.md`（role→`actor_id`）+ `_cast.md`（嵌 `_actors/actor_XXXX/*.jpg`）选定，**该选角图 = 长相权威源**。本阶段**不再问 / 不再写**眉/眼/鼻/唇/轮廓/痣/疤等五官——只确认该角色已分配 actor（未分配则提示用户去 webapp 选角）。文字仅保留「角色识别标签」（≤30 字造型可辨项，见 #6/#10），职责＝多人 shot 绑定消歧，**不重建五官**。
6. **服饰配色 + 发型 + 妆容（必锁）**：主色 + 辅色 + 点缀（**自然色名零 hex**，如 月白 / 玄青 / 暗金）｜ 形制（直裰 / 劲装 / 战甲…）｜ 发型发色 ｜ **妆容**（素颜 / 淡妆 / 伤妆 / 泪妆 / 血污 / 病气妆…，可标随剧情变化态，新增 2026-06-18）。**服饰要有质感、角色间颜色彼此区别**（避免撞色、强化辨识；地位高者用规制元素显尊贵）。**发型须贴合角色身份/职业有可辨差异**（习武者高束利落束发/武冠；文人/病弱者半束略散；妇人发髻），不一刀切（2026-06-18 follow-up）。
7. **说话风格模板（必锁，声口锚）**：✅ 话少低沉 ｜ 嚣张短句 ｜ 冰冷利落 ｜ 文绉绉端方 ｜ 市井油滑 ｜ 阴柔尖细——一句典型口吻示范。
8. **标志小动作（加分，可空）**：摩挲扳指 / 折扇轻敲 / 抬眼斜睨…（写进锁定描述符 #7，做记忆点 + 表演锚）。
9. **voice_id 选定（全剧锁定 · 同角色复用）**：给该角色一个稳定 id（如 `male-cold-01`）+ 声线 / 语速 / 口音 / 参考演员——**全剧 / 跨集只用这一个 id**，不同镜只改情绪 / 语速、不换音色（音色一致性铁律，rule 12.4-H）。

**C. 全剧风格**
10. **全剧调色 / 情绪光基调**：冷峻青灰 ｜ 暖金宫廷 ｜ 阴郁夜雨…（落 style_guide.md）。
11. **场景清单**：哪些地点 ≥2 镜复用 → 立场景档（其余 shot 级 inline）。

## 3. 生成步骤

按下列顺序生成 `2_世界观人设/` 全部产物。参考图（角色 ref 图 / turntable / 场景立绘）在本阶段一次生成、锁定后供后续所有 shot 复用。

1. **`world.md`** — 写：① bg 代号场景清单（每地点一个 `bg{N}` 代号 + 一句话）；② 力量体系（**含藏锋规则**：明文写"无金光 / 无瞳光 / 无外放威能，力量只以体内暖流 + 微表情 + 表演体现"，防 AI 乱加特效）；③ 势力划分（各方立场 + 与主角关系）；④ 修炼境界 / lore（如赛道需要）。
2. **`characters/{角色}/{角色}.md`** — 每个具名角色一份人物卡（folder 名 = 内层 md 名，rule 12.9）。含：基础标识（定位）+ 服化（发型 / **妆容** / 服饰 / 道具，自然色名零 hex）+ **声口模板** + 标志动作 + 性格动机 + **角色识别标签**（≤30 字造型可辨项，无五官）+ **4s 建立视频块**（turntable，rule 12.5；2026-06-27 压到 4s·4 秒内完成正→侧→背）——**每个命名人形角色一律要、不分主配、不分出场方式**（2026-06-27 统一建立模式 amendment）；有选角脸的吃选角脸、仅有自身立绘的轻量配角以自身 `cN_名字.png` 作参考脸。**脸不写文字五官**——长相由参考脸（选角图 / 自身立绘）承载（2026-06-18 选角供脸）。**非人形实体（系统/UI）例外、无 turntable。** 模板见 §4。
3. **`scenes/{场景}/{场景}.md`** — 每个复用场景一份场景档。含：场景定位 + 锁定描述符（方位 / 空间结构 / 默认光源时辰 / 一句话锁定，自然色名零 hex）+ **机位 / 方位**说明 + **bg 子代号**（对齐 world.md 的 `bg{N}`）+ Seedream 立绘 prompt 段。模板见 §4。
4. **`casting.md`** — voice_id 总表：每个具名角色一行（角色 / voice_id / 声线 / 语速 / 口音 / 参考），全剧锁定、同角色复用。模板见 §4。
5. **`style_guide.md`** — 全剧统一：调色基调（自然色名）+ 情绪光映射 + 正向关键词 + **负向锁定块**（人脸变形 / 五官漂移 / 多余发光 / 画面文字 / 畸形肢体 / 夸张金光 / 现代服饰…，全剧固定、各档复用）+ 渲染样式串。

> ref 图 / turntable / 场景图在本阶段生成并锁定 = 后续阶段 5–6 每个 shot 的 `参考:` 行只引用这里的 handle，不再重描。

## 6. 避坑

- **人脸靠选角图锁、服装靠人物卡复制锁**（2026-06-18 选角供脸）：脸的一致性来自每个 shot 上传同一角色的选角脸图 / turntable.mp4（同一 actor）；造型一致性来自 shot 阶段不重描、只 byte-identical 粘贴「角色识别标签」。漂移根因＝没传对参考图 或 没复制锁定串。**不再写 face-differentiator 五官 token**——多人 shot 防串脸靠「参考图↔位置↔识别标签」绑定（rule 12.4-B 新契约）。
- **藏锋规则写进力量体系**：world.md 明文"无金光 / 无瞳光 / 无外放威能，力量靠体内暖流 + 表演"，否则 AI 默认给修真角色乱加金光 / 瞳光。眼神变化只靠表演，不写发光特效。
- **色彩用自然中文色名（零 hex）**：hex 对 Kling / Seedance / Seedream 不可解析、只是噪声；删 hex 字段，用"月白 / 玄青 / 暗金"等色名（rule 1b）。
- **声口模板必锁**：不锁说话风格 → 跨集台词声口雷同 / 串味；每角色定死一句典型口吻 + 禁忌腔。

## 8. 机制

- **每次 update 默认复核**：任何对本阶段产物（world / 人物卡 / 场景档 / casting / style_guide）的改动或重生，默认重跑 `ai_videos__格式契约`（受影响范围）+ 在 `specs/ai_video/{name}/changelog.md` 记一条。改了某角色锁定串 → 必须同步该角色已被引用的下游 shot（否则跨镜漂移）。
- **反馈→进化**：每条实战反馈（如某 voice 串味、某面貌仍撞脸）→ surgical 更新本 playbook / `ai_videos__格式契约` 准则 / `agent_refs/project/ai_video.md`，附来源（角色 / 镜号 / 反馈引用）。流程越用越强。
