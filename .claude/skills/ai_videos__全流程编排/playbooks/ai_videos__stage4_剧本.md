# 阶段 4 · 单集文学剧本（台词） — playbook

> 流程位置：阶段 3（分集大纲）→ **阶段 4（本步）** → 阶段 5（分镜运镜）。
> 产出到此即可让台词「像真人说话」——这一步是区分「机械文案」和「真人台词」的分水岭。
> 交互默认 **interactive**：先用多选题问用户、用答案细化再生成（用户可说「这段自动」跳过提问）。

---

## 1. 何时 / 输入

- **何时**：阶段 3 大纲已过 QC（`剧情连贯` + `全剧序列` blocker 清零），开始把某集（或某批集）写成文学剧本。
- **输入**：
  - `3_大纲/arc_outline.md`（本集「上集承接 + 本集核心 + 下集伏笔」、起承转钩四段镜数/时长预算、本集钩尾类型）。
  - `2_世界观人设/characters/{角色}/{角色}.md` 人物卡（**声口模板** = 身份/性格/说话风格/标志口头禅 + voice_id；**`## 人物灵魂`** = 动机根因/目标/needs/desire/矛盾/恐惧/创伤/此刻决策逻辑/性格/成长轨迹）。
  - `2_世界观人设/relationships.md`（人物网 = 各角色社会/精神/情感关系 + 与当前主题关系）。
  - `2_世界观人设/world.md`（力量体系/系统设定/势力，台词不得越界引入未定义设定）。
- **创作前必读（rule 12.11-C）**：写每句台词 / 每段动作前，先读透该角色 `## 人物灵魂` + 与对手戏角色的 `relationships.md` 关系——台词与选择必须从「这个人此刻的动机/矛盾/恐惧/关系」长出来，不为爽点硬凑。写完回看：这句话、这个选择对得上他的灵魂与关系网吗？对不上就改台词/情节，或（人在成长时）回阶段 2 更新该角色成长轨迹。

---

## 2. 内建提问清单（AskUserQuestion 多选，标默认）

> 重要阶段、**多问**。每题给 2–4 选项，**默认项加「（默认·推荐）」**。用户答案细化后再生成。

1. **本集起承转钩拆段**：把 arc_outline 本集核心拆成「起（钩子）/ 承（铺垫）/ 转（小反转）/ 钩尾」四段，确认每段对应的剧情 beat 与时长占比（默认沿用 arc_outline 的镜数/时长预算）。
2. **台词口语化强度**：① **强口语（默认·推荐）**：脱口而出 / 短句 / 语气词 / 停顿；② 中等口语；③ 偏书面。
3. **内心独白（OS）用量**：① **仅关键节点（默认·推荐）**（转折/隐忍/打脸前蓄力）；② 较多；③ 几乎不用。
4. **系统 UI 文字风格**：**鎏金对话框 · 极简冷冷 · 仅主角可见 · 【】选项按钮（默认·推荐）**；或其它（用户自定）。
5. **三类台词标注规则确认**：确认按 ①口头对白 ②内心独白 ③系统 UI 文字 三类逐行标注（默认采用，规则见 §3）。

---

## 3. 生成步骤

为本集（或每集）生成两份文件：

### script.md — 文学分镜脚本
逐镜一行表格/块，字段：`镜号 | 时长 | 场景代号 | 画面动作描述 | 台词（三类标注）`。
- 画面动作描述 = 谁在哪、做什么、情绪氛围（不写机位/运镜，那是阶段 5）。
- 台词逐行打三类标签。

### dialogue.md — 纯台词
只留台词，按镜号顺序排，每行 `角色: "台词" (语气情感注释)`，方便配音/通读节奏。

### 台词三类（逐行必标）
| 类 | 标注 | 表演契约 |
|---|---|---|
| ① **口头对白** | `[对白]` | 出声、**有口型**、口语短句 |
| ② **内心独白** | `[OS]` | **嘴唇不动**、心底想法、情绪起伏（仅靠表情/眼神演） |
| ③ **系统 UI 文字** | `[系统]` | **仅主角可见**、极简无人称、冷冽（鎏金对话框 · 【】选项） |

---

## 4. 台词黄金规则

- **冲突/嘲讽**：多用**短问句 + 语气助词**（"区区废物，也敢大放厥词？行啊——"）。
- **隐忍内心戏（OS）**：带**不甘 / 疑惑 / 隐忍**，恨里有由、护里有怯。
- **旁白 / 系统**：压缩、**去连接词**，极简无人称、冷冽。
- **删**：书面破折号堆砌、冗长修饰、对仗工整的「念稿腔」、自指用第三人称（说自己就用「我」）。
- **白话铁律（台词大师 D1b）**：一律现代白话、**禁古语**（吾/汝/乃/之乎者也/矣/焉/岂/安能/莫非…耶 等），仙侠古装也不例外；身份/年代差异只靠用词分寸、语气、称谓体现，不靠说文言。

## 5. 输出物 + 模板

落点：`ai_videos/{name}/4_剧本/episodes/epNN/{script.md, dialogue.md}`。

**script.md 单镜模板（copy-ready）：**

```markdown
### 镜 {NN}
- 时长: {3–15}s
- 场景: {bg代号}
- 画面动作: {谁在哪、做什么、情绪氛围}
- 台词:
  - [对白] {角色}: "{口语短句}"
  - [OS] {角色}: "{心底想法}"
  - [系统]: "{极简冷冽提示}"
- 情绪氛围: {本镜情绪基调}
```

**dialogue.md 行格式（copy-ready）：**

```markdown
## 镜 {NN}
{角色}: "{台词}" (语气情感注释)
{角色}: "{台词}" (内心独白·不甘)
系统: "{提示}" (冷冽·仅主角可见)
```

## 6. QC 关卡（blocker 清零才进阶段 5）

跑 **`ai_videos__台词大师`**，逐行过：
- **说人话 / 白话（禁古语）/ 因果 / 信息量 / 声口 / 情绪**（D1–D6）。
- **节奏**：一镜总台词字数 **÷ 时长(秒) ≤ 5**（目标 ≈ 4），念不完先精简台词，要不要加时长/拆镜由 `时长节奏` 仲裁。

发现不合格 → 当场 surgical 改写，三处同步（shot 台词 / dialogue.md / script.md）。**blocker 必须清零**方可进阶段 5。

---

## 7. 避坑

- **严禁跳过剧本直接写 prompt**——这步是区分「机械文案」和「真人台词」的关键；没有文学剧本兜底，prompt 里的对白必然空、假、念稿腔。
- 台词**去书面长句 / 对仗 / 破折号堆砌**；冲突用短问句 + 语气助词，OS 带不甘隐忍，系统句去连接词。

---

## 8. 衔接

- **上游**：阶段 3 分集大纲（`arc_outline.md` 提供四段结构 + 钩尾）。
- **下游**：阶段 5 分镜运镜（按 script.md 的镜号/画面动作/台词三类拆运镜机位）。

---

## 9. 机制（贯穿）

- **每次 update 默认复核**：任何对 script.md / dialogue.md 的改动/重生，默认跑受影响范围的 `ai_videos__台词大师`（必要时连带相邻镜连贯性 check）+ 在 `specs/ai_video/{name}/changelog.md` 记一条。
- **反馈→进化**：用户每条实战台词反馈 → surgical 更新本 playbook / `台词大师` 准则表 / agent_refs + 记一条「教训」（带来源镜号/反馈引用）。流程越用越强。
