# 阶段 5 · 分镜运镜设计 — playbook

> AI 短剧全流程编排的第 5 步。把每集**文学剧本**拆成**镜头清单 + 每镜的运镜设计字段**（构图机位 / 动态运镜 / 走位动作分秒 / 光影色调）。
> 全局设计见同目录 `BLUEPRINT.md`（重点 §4 锁定默认 / §5 阶段5 / §6 三大机制）；编排循环见 `../SKILL.md`；shotNN.md 结构见 `agent_refs/project/ai_video.md` rule 5（三段式）+ rule 12.4（字段顺序与必填矩阵）。
>
> ⚠ **阶段 5、6 产物合一在同一个 `shots/shotNN/shotNN.md`**。本阶段**只写运镜设计字段**——`镜头:`（构图机位）/ `走位:` / `动作:`（分秒走位动作）/ `光线 / 色调:` / `节奏:`；阶段 6 再补 `参考/角色/情节/场景/台词/渲染样式/比例/负向`、校五层 prompt 契约、出汇编。**本阶段不要碰阶段 6 的字段，也不要把 shot 当成"半成品 prompt"草率塞满。**

---

## 1. 何时 / 输入

- **何时**：某集 `script.md` 已定稿、过了阶段 4 QC（台词大师），要把它拆成镜头并设计运镜。或恢复一个有剧本、尚无 `5_6_分镜与prompt/episodes/epNN/shotlist.md` 的集。
- **输入**：
  - `ai_videos/{name}/4_剧本/episodes/epNN/script.md`（每集；镜号|时长|场景|画面动作|台词三类标注）——主输入。
  - `ai_videos/{name}/2_世界观人设/characters/{角色}/{角色}.md` 人物卡（体型/身份/性格 → 决定俯仰机位、反应方式、走位）。
  - `ai_videos/{name}/2_世界观人设/scenes/{场景}/…` 场景档（机位/方位/朝向 plate → 决定轴线、站位一侧、进出画）。
  - `style_guide.md`（情绪光表 / 镜头词典 / 调色）。
- **目标**：每集产出 `shotlist.md` + 每镜 `shotNN.md` 的运镜设计字段，做到**先情绪后机位、运镜配动机、走位动作配分秒、光影随情绪**。

## 2. 内建提问清单（interactive：先问后做）

用 `AskUserQuestion`（多选题）逐题问；每题列选项并把**已锁默认**标 `(推荐)`（默认源自 BLUEPRINT §4）。
**用户可随时说"这段自动"/"用默认"** → 跳过提问，直接采用推荐项生成。

| # | 题目 | 选项（`(推荐)`=已锁默认） |
|---|---|---|
| Q1 | **每场景连贯镜序** | 按 script 镜号顺拆`(推荐)` / 重新切分（合并装饰镜·拆超长 beat） / 用户指定镜数 |
| Q2 | **每镜情绪目的（先定情绪）** | 逐镜先标情绪目的再出机位`(推荐)` / 用户逐镜给情绪 — 这是本阶段命门：**机位由情绪推导，不可反过来先堆运镜** |
| Q3 | **走位与相对朝向** | 按场景方位 plate 锁同侧 + 写死相对朝向`(推荐)` / 用户指定关键镜站位 |
| Q4 | **单镜时长** | 3–15s 按 beat 给`(推荐，由时长节奏裁决)` / 固定 5–8s 快切 / 用户逐镜给 |
| Q5 | **光影色调基调** | 跟 style_guide 情绪光表 + 场景锁定色`(推荐)` / 用户指定本集冷暖基调 / 回忆段做旧滤镜 |

> Q2 是本阶段的命门，无法用默认糊弄成"堆运镜名词"——**每镜必须先有一句情绪目的**（压抑 / 震动 / 杀机 / 对峙 / 嚣张…），机位从 §3 映射表推导。若用户"这段自动"，仍须逐镜先写情绪目的、再据映射表出机位，不得直接罗列推拉摇移。

## 3. 情绪 → 运镜映射表（本阶段核心：先情绪、后机位）

**铁律**：每个运镜都配一个**情绪目的**；运镜是情绪的手段，不是装饰。**禁止堆砌运镜名词**（"推+拉+摇+移+升降"塞满一镜）——一镜通常只配 1 个主运镜 + 1 个情绪目的。运镜要**缓、控速、稳、不抖**（保 Kling 稳定性）。

| 情绪 / 戏剧目的 | 构图机位（景别 + 角度） | 动态运镜（含速度） | 为什么 |
|---|---|---|---|
| **压抑 / 孤寂 / 渺小** | 低位远景 / 大全景，人物占画很小，环境压上方 | 缓慢后拉（slow pull-back），人物渐显渺小 | 拉开空间感、强化"被环境/命运压住" |
| **内心震动 / 顿悟 / 反应** | 面部特写 / 大特写，眉眼为焦点，背景浅景深虚化 | 极缓微推（slow push-in）至眉眼，定格瞬间 | 放大微表情、把观众钉在角色内心那一下 |
| **暗藏杀机 / 阴谋 / 反派蓄势** | 上下分割双区构图 / 前后景错位；反派处暗部剪影、半脸阴影 | 固定机位或极缓侧移，暗部留住悬念 | 明暗分区藏信息、剪影=只露轮廓不泄底（藏锋·无外放） |
| **对峙 / 权力压制** | 保持轴线（180°线不越）；**主角在下/前景、对方在上/高座**，仰俯差固定 | 极缓推近主角面部 / 双方反打稳机位 | 高低差=权力关系；轴线锁住防左右乱跳 |
| **嚣张 / 挑衅 / 正面对立** | 过肩反打（over-shoulder），双方视线相对、各占一边 | 反打稳机位，关键狠话处微推说话人 | 反打给对抗张力——**但禁背对**（见 §7 避坑） |
| **奔逃 / 追击 / 动势** | 中景 / 全景跟随，留运动前方空间 | 跟移贴步伐律动、小幅起伏（手持感但稳） | 运动方向顺承、观众跟着动 |
| **抒情 / 怅然 / 留白** | 中景 / 中近景，构图留白 | 极缓升降或刻意凝滞固定机位（写明动机） | 给情绪呼吸；静止只在"刻意凝滞/留白"用、写明动机 |
| **冷暖对冲 / 转场收束** | 镜内由暗向门外天光 | 台词收尾轻推门外天光，做冷暖对冲 | 用光做情绪转折与段落收口 |

> 用法：先给本镜一句情绪目的，再从本表选机位 + 运镜填进 `镜头:`。摒弃**死板固定机位**与**呆板对称构图**；纯静止锁机位只在"刻意凝滞/留白"且写明动机时用（2026-06-17 EP2 全镜优化教训）。关键情绪/反转/眼神瞬间用缓慢推近至面部/眼睛特写（ai_video.md rule 12.4 nvdi 030）。

## 4. 生成步骤

1. **读**：本 playbook + `agent_refs/{validation}/general.md`+`ai_video.md` + `script.md`（本集）+ 人物卡 + 场景档 + style_guide + BLUEPRINT §4。
2. **问**：按 §2 清单用 `AskUserQuestion` 多选题逐题问（除非用户说"这段自动"）。
3. **拆镜出 shotlist**：按场景把 script 切成连贯镜序，生成 `shotlist.md` 表：`镜号 | 内容 | 出场角色 | 景别+运动 | 时长 | 标记`（标记 = 黄金钩/反转/cliff/回忆/系统流）。每镜**先标一句情绪目的**，再据 §3 映射表填"景别+运动"。底部加 `时长合计` 行。
4. **逐镜写运镜设计字段**（落 `shots/shotNN/shotNN.md`，folder-per-shot，rule 12.9）：本阶段只写这几个字段——
   - `镜头:`（**构图机位含目的**：景别+角度+动态运镜+速度，并注明该机位服务的情绪目的；人物近景/特写注"背景浅景深虚化柔焦、主体清晰"）。
   - `走位:`（在画每人**具体角色名**、相对位置 + 相对朝向 + 视线落点；同场景同一人固定一侧；同角色每镜只具名一次，必要时写"全画面共 N 人各一名"）。
   - `动作:`（**0–Ns 分段 timed beats**：每段 macro 动作 + 微表情 + 身体朝向&视线 + 氛围微动；beats 铺满 shot 时长；配角有合理反应、不干站。**末拍写清定格姿态**——它＝交给下一镜的承接帧画面，见下条）。
   - `光线 / 色调:`（随情绪给光，对齐场景/角色锁定色名、零 hex；回忆镜上做旧滤镜；眼里不加发光特效）。
   - `节奏:`（visual-only 的节奏注：快切/凝滞/渐快等，配合时长）。
4b. **逐对判跨镜首帧承接、写 `## Shot context` 的 `衔接:` 字段**（ai_video.md 2026-06-21；判定准则归 `运镜` M8）：相邻 A→B 满足①同场景/同bg ②机位连续或渐变(同轴推/拉/移·match cut、非越轴/正反打/景别跳切) ③动作或情绪不间断 → **承接**（`衔接: 承接 shot{NN} 末帧（首帧＝上一镜末帧）`，B 出片时截 A 成片末帧作 B 首帧上传、`动作:`首拍从 A 末拍姿态续起）；任一不满足 → **硬切（默认）**（`衔接: 硬切（独立首帧）`）。换场/时间跳/回忆进出/越轴/正反打/景别跳切一律硬切。每集首镜＝硬切。承接镜的 `参考:`/`Reference uploads` 落点由阶段 6 补（见 stage6 playbook §3）。**同时给被承接的上一镜（交接源）写 `尾帧锁定:` 字段**（ai_video.md (G)）：该镜末帧既被下镜用作首帧，重生成它时须以其 `shotNN_lastframe.png` 作尾帧锁定、保末帧不变，避免「改一镜要整条承接链重生」。
5. **过 QC**：按 §6 跑五道审查关卡；blocker 清零才进阶段 6。
6. **记审计 + 衔接**：审计写 `.audit/adhoc_agents/{date}/{task_id}/`（含 `pre_reading_consulted`）；进阶段 6（同一 shotNN.md 补剩余字段 + 五层契约）。

## 7. 避坑

- **先情绪后机位、不堆运镜名词**：每镜先一句情绪目的，再据 §3 表出 1 个主运镜；禁"推+拉+摇+移"塞满一镜的炫技堆砌。
- **A 不背对 B 说话**：在画对白须写死说话人正脸朝对象。⚠ 慎用"偏头"——模型常把"偏头"渲成"转头背对"（shot09 教训）。对**正在离去/背对的对象**说话时，写"整个人转身正面朝其离去方向、目光锁背影"，绝不让说话人背对镜头/对象。
- **同角色每镜只具名一次**：多次具名 + 多位置短语会被模型渲成两个同人（shot03 出现两个裴知秋）。首次在 `走位:` 定位，其余用称谓；必要时写"全画面共 N 人各一名、互不重复"。
- **动作配时间窗、配角有反应、不干站**：`动作:` 拆 0–Ns 分段 beat 铺满时长；每个在画配角对眼前事件给合理反应（被偷袭→惊呼/躲闪，不呆站）；删纯装饰的"主角站着"镜或并镜。
- **内心独白/失神镜把视线写死一个具体落点**（如"目光垂落身前地面、不乱飘"），否则眼神乱瞟（shot10 教训）；内心独白不对口型。
- **回忆镜光线一眼可辨**（做旧泛黄+低饱和暖褐+暗角+胶片颗粒），同段回忆滤镜彼此一致。
- **眼睛不写发光**：能量入体改"眉心一缕微光内敛"，眼变化靠表演（藏锋·无外放）。

## 9. 机制（贯穿，见 BLUEPRINT §6）

- **每次 update 默认复核**：日后任何对 shotlist.md / shotNN.md 运镜字段的改动/重生，默认跑受影响范围的 `ai_videos__审查总编排`（至少本阶段五道）+ 在 `specs/ai_video/{name}/changelog.md` 记一条（改了什么、连带影响哪些相邻镜/下游字段）。改运镜/走位后默认做相邻镜连贯性 check（轴线/视线/景别/运动/站位顺承）。
- **反馈 → 进化**：用户每条实战反馈 → surgical 更新本 playbook / 对应审查 skill（准则表加一行）/ `agent_refs` + 记一条"教训"（带来源镜号/反馈引用）。**流程越用越强**。
