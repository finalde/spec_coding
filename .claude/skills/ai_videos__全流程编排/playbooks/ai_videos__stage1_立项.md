# 阶段 1 · 核心创意立项 — playbook

> AI 短剧全流程编排的第 1 步。把一个脑洞落成可执行的**立项策划单 concept.md**。
> 全局设计见同目录 `BLUEPRINT.md`（重点 §4 锁定默认 / §5 阶段1 / §6 三大机制）；编排循环见 `../SKILL.md`。

---

## 1. 何时 / 输入

- **何时**：一部新短剧的起点（脑洞起步），上游无产物。或恢复一个只有想法、尚无 `1_立项/concept.md` 的项目。
- **输入**：用户的想法（一句话脑洞、一段设定、一个赛道偏好，皆可——哪怕只有"我想做个古风复仇短剧"）。
- **目标**：把模糊脑洞固化成一张含**主角诉求 + 反派动机 + 长线伏笔**的立项策划单。

---

## 2. 内建提问清单（interactive：先问后做）

用 `AskUserQuestion`（多选题）逐题问用户；每题列出选项并把**已锁默认**标 `(推荐)`（默认源自 BLUEPRINT §4）。
**用户可随时说"这段自动"/"用默认"** → 跳过提问，直接采用推荐项生成。

| # | 题目 | 选项（`(推荐)`=已锁默认） |
|---|---|---|
| Q1 | **赛道题材** | 古风仙侠复仇`(推荐)` / 都市重生逆袭 / 古风宅斗 / 悬疑脑洞 / 其它(用户填) |
| Q2 | **单集时长** | 90–120s`(推荐)` / 60–90s / 120–180s |
| Q3 | **总集数 / 篇幅** | 自适应`(推荐，立项时定)` / 短篇(≤12集) / 中篇(12–40集) / 长篇(40+集) |
| Q4 | **目标平台** | 红果/番茄`(推荐)` / 抖音·快手 / 海外 YouTube·TikTok / 全平台9:16母版`(推荐)` |
| Q5 | **核心爽点主线** | 自由填：**一句话概括全剧核心冲突**（如"被灭门的少主藏锋十年、逐个反杀仇敌"）。无默认，必须给。 |
| Q6 | **核心人物对立** | 自由填：**主角 + 反派 + 关键配角 + 暗线人物**各一句（谁要什么、谁挡路、谁是暗钉）。无默认，必须给。 |
| Q7 | **每集硬性钩子规则** | 每集结尾必预埋危机/伏笔`(推荐)` / 钩尾类型轮换(危机·情感·实力·信息) / 用户自定规则 |

> Q5、Q6 是立项的命门，无法用默认糊弄——必须从用户想法里抽出或追问到。若用户"这段自动"，仍须用其脑洞推导出主角诉求/反派动机/长线伏笔三件套，不得留空。

---

## 3. 生成步骤

1. **读**：本 playbook + 用户想法 + `agent_refs/project/ai_video.md` + BLUEPRINT §4 锁定默认。
2. **问**：按 §2 清单用 `AskUserQuestion` 多选题逐题问（除非用户说"这段自动"）。
3. **抽三件套**：从答案里明确写死 ① 主角诉求（要什么）② 反派动机（为何与主角对立）③ 长线伏笔/暗线（玉佩/黑影式钩子，贯穿全剧的悬念）。
4. **填模板**：用答案 + 锁定默认填 §4 的 concept.md 模板，落 `ai_videos/{name}/1_立项/concept.md`（`{name}` 为 pinyin/英文）。
5. **过 QC**：按 §5 人工确认；不齐回到第 3 步补。
6. **记审计 + 衔接**：审计写 `.audit/adhoc_agents/{date}/{task_id}/`；进阶段 2。

---

## 4. 输出物 + 模板

- **产物**：`ai_videos/{name}/1_立项/concept.md`（内容全中文，路径英文/pinyin）。
- **copy-ready 模板**（直接复制填空）：

```markdown
# 立项策划单 · {剧名（中文）}

## 赛道题材
{古风仙侠复仇 / …}

## 全剧正向立意（一句话）+ 主角价值落点
{全剧积极向上的立意一句话 + 主角最终的正向价值兑现，如：藏锋十年非为泄愤，而是护住该护的人、还天下一个公道。**红果 2026-04-07 漫剧新规硬要求**——见 ai_video.md §14.2}

## 核心爽点主线（一句话）
{一句话概括全剧核心冲突，如：被灭门的少主藏锋十年、逐个反杀仇敌}

## AI 强项 beat 偏好（扬长避短）
{优先编排 AI 擅长的奇观/打斗/变身/爽点反转当钩子与封面候选；情感戏靠台词+分镜+光色+配乐承载，不靠 AI 硬演细腻微表情——见 ai_video.md §14.1}

## 核心人物对立
- **主角**：{名}——诉求：{要什么}；藏锋设定：{力量/身份如何隐藏}
- **反派**：{名}——动机：{为何与主角对立、要什么}
- **关键配角**：{名}——立场：{助主角/中立摇摆/工具人}
- **暗线人物**：{名}——埋点：{何时浮现、揭什么秘密}

## 单集时长 · 总集数
- 单集时长：{90–120s}
- 总集数 / 篇幅：{自适应 / N 集}
- 目标平台：{红果+抖音+海外，9:16 母版}

## 每集硬钩规则
{每集结尾必预埋危机/伏笔；钩尾类型库：危机/杀机 · 情感/身世 · 实力/藏锋将显 · 信息/秘密——每集择一}

## 长线伏笔 · 暗线清单
- {伏笔1：如"玉佩"——首集亮相、第N集揭真身}
- {伏笔2：如"黑影"——贯穿全剧的幕后推手}
- {…}
```

## 5. QC 关卡（人工确认）

**关卡 = 人工确认**（本阶段无审查 skill，由用户/编排者点头）。**三件套齐全才过关、才进阶段 2**：

- [ ] **主角诉求**明确（主角要什么、为何而战）。
- [ ] **反派动机**明确（反派要什么、为何与主角对立——不是工具人）。
- [ ] **长线伏笔/暗线**存在（玉佩/黑影式贯穿全剧的悬念钩子，非临时编造）。
- [ ] **全剧正向立意 + 主角价值落点**写死（红果新规硬门槛——通篇炫恶/泄愤无收束/三观崩坏不过关，见 ai_video.md §14.2）。

任一缺失 = blocker，回 §3 第 3 步补齐，**不得带病进阶段 2**。

---

## 6. 避坑

- **杜绝模糊脑洞**：不能停在"古风复仇爽剧"这种空设定就开工——必须落到具体的主角诉求、反派动机、隐藏长线伏笔。
- **反派不是工具人**：反派动机要能自洽、能撑起全剧对抗，否则后续每集冲突无源、爽点泄气。
- **必须有长线伏笔**：缺了贯穿全剧的暗线（玉佩/黑影式），后续分镜情节会割裂、各集像散装短片、无追剧驱动力。
- **爽点主线要一句话说清**：说不清一句话主线 = 立项不成立，后面大纲/剧本会失焦。

---

## 7. 衔接

- **上游**：无（全流程起点）。
- **下游**：阶段 2 · 世界观 + 锁定人设（`ai_videos__stage2_世界观人设`）——concept.md 的人物对立/赛道/藏锋设定是世界观与人设卡的输入。

---

## 8. 机制（贯穿，见 BLUEPRINT §6）

- **每次 update 默认复核**：日后任何对 concept.md 的改动/重生，默认跑受影响范围的 `ai_videos__审查总编排` + 在 `specs/ai_video/{name}/changelog.md` 记一条（改了什么、连带影响哪些下游产物）。
- **反馈 → 进化**：用户每条实战反馈 → surgical 更新本 playbook / 对应审查 skill / `agent_refs` + 记一条"教训"（带来源/反馈引用）。**流程越用越强**。
