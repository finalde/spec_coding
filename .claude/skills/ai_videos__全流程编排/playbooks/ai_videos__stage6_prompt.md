## 2. 内建提问清单（AskUserQuestion 多选，标默认）

> 锁定项已在阶段 1/2 定稿，本步主要**确认**，不重开。每题默认项加「（默认·推荐）」。

1. **目标生成模型**：① **即梦 Seedance + 可灵 Kling（默认·推荐）**（model-agnostic 同一 schema，文件名不区分 variant）；② 仅 Seedance；③ 仅 Kling；④ 加 Sora/Veo 等。
2. **渲染风格**：**影视级真人写实仙侠 · 全剧统一渲染样式串（默认·推荐）**；或项目自定（须写进 style_guide 并全场景统一、记 divergence）。
3. **负面词块确认**：确认沿用全剧固定负面词块（默认采用，内容见 §3 层 5）——人脸变形 / 五官漂移 / 多余发光特效 / 画面文字 / 畸形肢体 / 夸张金光 / 现代服饰。
4. **字幕策略**：**prompt 零字幕 · 后期自加（默认·推荐）**（prompt 内严禁任何字幕信息，字幕由用户后期叠加）；无其它合法选项（零字幕是硬契约，rule 12.4 v2）。

## 3. 五层标准化 Prompt 模板（核心）

把阶段 5 的运镜设计**重组并补全**为下面的五层结构。五层是**阅读 / 编写组织法**，落地仍是 rule 12.4 字段级 schema（参考/角色/情节/场景/镜头/走位/动作/台词/光线/节奏/渲染样式/比例/时长）——五层是这些字段的分组视图，缺任一字段 = stage-6 校验失败。

| 层 | 内容 | 对应 rule 12.4 字段 |
|---|---|---|
| 层1 · 场景 + bg 代号 | 固定 bg 代号 + 一句话场景锁定 | `参考:`（场景 token）+ `场景:` |
| 层2 · 锁定角色 | 复制人物卡锁定描述符 byte-identical；剪影/背影配角只写轮廓 | `参考:`（人物 token）+ `角色:` |
| 层3 · 剧情 + 分镜动作 | 情节承接 / 镜头 / 0–Xs 分段动作 / 台词标注（画面无字幕） | `情节:` `镜头:` `走位:` `动作:` `台词:` |
| 层4 · 光影色调 + 氛围 | 随情绪的光线色调；禁人物自发光、无多余金光、藏锋无外放 | `光线 / 色调:` `节奏:` |
| 层5 · 渲染统一参数 + 负面词块 | 全剧统一渲染串 + **负面词固定块（必填）** | `渲染样式:`（+ 负面词块） |
| — | 单镜时长（阶段 5 已定，本步透传） | `时长:` `比例:` |

**copy-ready 五层模板（落进 `## 视频 prompt` 的 ```text 块）：**

```text
{NN}集{NN}镜视
参考: `{人物1}=>, {人物2}=>, {bg代号}=>`
角色: `{人物1}: {角色识别标签·byte-identical 自人物卡第8行，造型可辨项无五官}; {剪影配角}: {只写从背面可见的轮廓·衣色/发型/体态，不给五官}`
角色识别 / 参考图: `{人物1}＝参考图(turntable/选角脸·见参考行); 多人物同框时逐个写死「参考图↔画面位置↔识别标签」绑定，如 画面左·{人物1}=参考图A·{标签}; 画面右·{人物2}=参考图B·{标签}（防 Seedance/Kling 串脸；脸由选角图承载，此处只做绑定消歧，不写五官，2026-06-18 选角供脸 rule 12.4-B）`
情节: `{承接上镜的小说原文/剧情·verbatim；具体角色名不用关系称谓}`
场景: `{bg代号} · {一句话场景锁定}`
镜头: `{景别} + {运动}（近景/特写注「背景浅景深虚化柔焦、主体清晰」；关键情绪瞬间用缓慢推近至面部）`
走位: `{每个在画人物的位置 + 朝向 + 相对位置·世界坐标系东南西北，不用画面左右；说话人朝向写死、视线落对方}`
动作: `0–{a}s … / {a}–{b}s … / {b}–{X}s …（分秒 beat，beats 之和 = 时长）`
台词:（画面不显示文字、仅供口型与配音参考；逐条↓）
· {说话人}〔{正常台词 / 内心独白}·{语气·口型注：正常台词→口型对{说话人}；内心独白/画外/系统→嘴唇不动、不对口型}〕：{台词正文·纯文本不加引号}
光线 / 色调: `{随情绪的光影色调；藏锋无外放、人物不自发光、无多余金光；眼变化靠表演、不加发光特效}`
节奏: `{慢 / 中 / 快 / 顿挫·纯视觉节奏}`
渲染样式: 影视级真人写实 · 电影级唯美仙侠 · 电影质感 · 竖屏 · 浅景深特写 · 逆光描边 · 东方修真古风 · 工笔质感肤色白皙通透 · 三庭五眼东方面孔 · 精致纹样场景 · 淡仙气氛围 · photorealism · 画面不烧任何字幕文字（台词后期另加）。
负面词: 人脸变形 / 五官漂移 / 多余发光特效 / 眼睛发光 / 瞳孔发光 / 画面文字 / 畸形肢体 / 多余手指 / 夸张金光 / 现代服饰 / 群众围观 / 卡通动画感。
比例: 9:16
时长: {3–15}秒
```

> 注：
> - **渲染样式串 + 负面词块全剧 byte-identical**——逐字复制 `style_guide.md`，每镜重贴同一串。
> - **`负面词:` 行为必填硬契约**（与 rule 12.4 v4 `负向:` 字段同源）；项目若把负向移到平台反向输入框（per ai_video.md nvdi 026），则负面词块挪到那里、prompt 块内可省 `负面词:` 行，但「要抑制什么」不变。
> - **每个字段值用反引号包裹**，帮模型分辨结构段边界。
> - **每镜自包含**：prompt body 严禁「承 shotNN / 见 shotNN / 下一镜」等跨镜引用（生成时每镜独立喂入）。
> - **字数**：视觉指令 body soft ≤ 2000 字 / hard ≤ 2500 字（`情节:` 不计入）。

## 4. 台词配音 prompt 块（每个说话镜必补）

每个有台词的 shot 在 `## 视频 prompt` 之后追加独立 `## 台词配音 prompt` 段（**每个发声单元各一个 ```text 块**）。三声线解耦、后期 mux。

```text
{NN}集{NN}镜 · 台词配音
角色: {说话人}
音色(锁定·全剧复用 · {voice_id}): {声线描述·取自角色 bible}
情绪: {本镜情绪}
语速: {语速·按时长目标定}
类型: {正常台词 / 内心独白 / 系统提示音}
台词: {台词正文·阿拉伯数字改口语中文}
时长目标: {秒·各句之和 ≤ shot 时长}
```

- **音色一致性铁律**：同一角色全剧 / 跨集复用**同一 `voice_id`**，不同镜只改情绪 / 语速、不换音色。voice_id 与角色 bible / casting.md 对齐。
- **音画解耦**：视频**不烧自动字幕、不依赖视频自带 TTS**；台词 MP3 单独生成，后期 `tools/mux_av.py` 把 video MP4 + 台词 MP3 +（可选）BGM 合成成片。
- **画外 / OS / 系统句**：类型标 `内心独白 / 系统提示音`，配套视频 prompt 的 `台词:` 标「嘴唇不动、不对口型」「严禁对到在画人物嘴上」。
- **默剧 / 静默镜**：省略本段（或标「无台词」并注环境音）。

## 5. 生成步骤

1. 读全部输入（阶段 5 的 shotNN.md + 人物卡锁定串/voice_id + 场景档 + style_guide + script/dialogue）。
2. **逐镜把运镜设计补全成五层 prompt**：层1 填 bg 代号 + 场景锁定；层2 byte-identical 复制锁定描述符（剪影只写轮廓）；层3 补情节承接 + 透传镜头/走位/动作分秒 + 台词标注；层4 补随情绪光影、藏锋无外放；层5 重贴全剧统一渲染串 + 负面词块；透传时长/比例。
3. **每个说话镜补 `## 台词配音 prompt` 块**（每发声单元一块），voice_id 对齐 bible。
3b. **带显著情绪 beat 的镜引用演技库**（§9b + 2026-06-18 演技库 amendment）：按 情绪大类+强度+时长 在 `_performances/` 匹配 top-N，选中条目把锁定文本块**融入（不照抄）**该镜 `动作:`/表情，并加 `表演库参考: perf_NNNN (情绪·强度·风格·载体) — 用于 <角色> <beat>` 标注行。webapp 可推荐候选 + 多选 + 一键重生（`POST /api/regen-shot-prompt`）。
4. **生成全集汇编 `all_shot_prompts.md`**：自动汇编自各 shot 源，含每镜视频 prompt +（说话镜的）台词配音 prompt，顶部注「只读快照——改各 shot 源后重新汇编」。
5. **人物出场定格字卡（rule 11d · 标准步骤）**：扫本集有无**重要角色首次登场**（主角/核心反派/关键长线配角；龙套·路人·系统不发卡，门槛见 ai_video.md 11d / style_guide §7）。有则 ① 该首登镜 `## Shot context` 加 `首登字卡:` 行（字卡文字**不入** `## 视频 prompt` ```text``` 块）；② 该集建 `episodes/epNN/intro_cards.md`（行式：`角色 | 首登shot | 定格点(秒) | 主名 | 副身份 | 字体/样式 | 位置 | 时长`，文案按角色 bible 拟、用户审）。同角色全剧仅首登发一次——本集若无重要新角色首登（如纯延续集）则**省此文件**。后期由 webapp「🪧 人物卡」(`POST /api/burn-intro-cards`) 按 intro_cards.md 定格亮相 + 烧名牌 → 生成 `shot{NN}.mp4`（落 shot 根目录、不覆盖 `renders/` 原片、二次烧录可覆盖）。

## 8. 避坑

- **特效严限「无外放威能 · 人物不自发光」**：力量靠体内暖流 + 表演，禁金光 / 瞳光 / 裂纹 / 威能光 show；眼神转锐靠瞳孔微缩 / 眼睑收紧 / 目光如刃（神态语言），**严禁写「眼睛发光 / 瞳孔发光 / 锐光 / 眼内光效」**（Kling 会字面渲染成射光）。
- **台词与画面剥离**：`台词:` 只留说话人 + 正文 + 类型 + 口型注，**画面零字幕**（禁字体 / 字号 / 位置 / 颜色 / 内嵌硬字幕 / 后期软字幕 / 字幕窗）；`台词:` 正文**绝不清空**（删了成片变哑），全员无口型镜用「画外配音 voice-over + 禁 lip-sync」而非删文。
- **剪影暗线只写轮廓禁五官**：黑影 / 暗线角色只给「纯黑剪影、按刀伫立、面目不可辨、不给辨识特征、不显形」，留待后续集显形。
- **多角色同框同角色只具名一次**：避免视频模型把同一人渲染成两个；≥2 个背影时逐一点明各自区分特征（衣色 / 发型）+「严禁雷同或都渲染成同一人」。

## 10. 机制（贯穿）

- **每次 update 默认复核**：任何对 shotNN.md / all_shot_prompts.md 的改动 / 重生，默认跑受影响范围的 `ai_videos__格式契约`（必要时连带相邻镜连贯性 check 与 `审查总编排`）+ 在 `specs/ai_video/{name}/changelog.md` 记一条。
- **反馈→进化**：用户每条实战 prompt 反馈 → surgical 更新本 playbook / `格式契约` 准则表 / agent_refs + 记一条「教训」（带来源镜号 / 反馈引用）。流程越用越强。
