# 阶段 3 · 分集大纲 playbook（arc_outline）

> 编排器 `ai_videos__全流程编排` 在第 3 阶段读本文件 inline 执行。interactive：先按下方提问清单用多选题问用户、用答案细化再生成。用户说"这段自动"则用默认值跳过提问。
> 上游=阶段 2（世界观+人设）；本阶段产出分集大纲；下游=阶段 4（文学剧本）。
> **⚠ 敏捷原则（2026-06-18，优先于下方任何"全剧一次切完"的默认）**：大纲**只定全剧大方向骨架 + 前几集（用户指定范围）的细纲**；其余各集**留松、只占位**（一行方向即可），推进到该集时（阶段 4–6）再细化，并随实拍 feedback 临时调整、不绑死。**不要一次性把全剧逐集剧情写满**。

---

## 1. 何时 / 输入

- **何时**：阶段 2 已过 QC（`2_世界观人设/` 齐全、`ai_videos__格式契约` blocker 清零），开始把全剧切成集、定每集骨架。
- **输入**：
  - `1_立项/concept.md`（赛道/总集数/单集时长/核心爽点主线/人物对立/硬钩规则/长线伏笔）。
  - `2_世界观人设/{world.md, characters/{角色}/…, scenes/…, casting.md, style_guide.md}`（力量体系·藏锋尺度/势力/锁定人设/voice_id/场景代号/canon）。
- 不读后续阶段产物。大纲只对齐 concept + world/人设 canon。

---

## 2. 内建提问清单（AskUserQuestion 多选，标默认）

`AskUserQuestion` 是 parent-only；本阶段 parent-direct，由编排器直接问。每题给选项并**标默认（推荐项）**：

1. **总集数切割方式** —
   - 默认：`三幕骨架 + 一行/集`（先按起/承/转·合分幕定季度拍点，再逐集落一句话方向纲领；前 3 集 72h 高热窗钩密度最高）。
   - 备选：均匀 N 集等切 / 按爽点峰反推集数 / 单元剧式。
2. **每集「上集承接点 + 本集核心事件 + 下集预埋伏笔」** —
   - 默认：`逐集三联标注`（每集必写承接/核心/伏笔三栏，串成情节链）。
   - 备选：仅强钩集详标、其余从略。
3. **起承转钩四段** —
   - 默认（**固定**）：`起=开场钩(3s 内抓人) / 承=铺垫(信息+情绪累积) / 转=反转爽点(逆袭/成长/识破/转机，不强调打脸) / 钩尾=悬念`。四段顺序与功能锁死，不改名不调序。
4. **每段分配镜数 + 总时长**（单集 90–120s 预算） —
   - 默认：`起 2–3 镜 / 承 4–6 镜 / 转 3–5 镜 / 钩尾 1–2 镜`，合计约 12–16 镜、总时长落在 90–120s（单镜 3–15s 按 beat，由 `ai_videos__时长节奏` 终裁）。
   - 备选：快切多镜（更碎）/ 长镜少镜（更稳）。
5. **每集钩尾类型**（四选一备选清单） —
   - 默认：`按 beat 择一`，从 `危机/杀机` · `情感/身世` · `实力/藏锋将显` · `信息/秘密` 四类里挑最贴当集结尾的一种。
   - 强钩集（72h 窗 + 章节级反转点）优先 `危机/杀机` 或 `实力/藏锋将显`。
6. **节奏规划**（压抑/冲突/悬念三态） —
   - 默认：`压抑段慢镜 / 冲突段快切 / 悬念结尾长暗镜`（受辱铺垫用慢镜累积压抑→打脸爽点用快切→钩尾用长暗镜留悬念）。
   - 备选：全程快切（强留存）/ 匀速。

> 答案逐项落进生成步骤；用户未答的用默认。

---

## 3. 生成步骤

1. 读 `concept.md` + `world.md`/人物卡，抽出：总集数、核心爽点主线、人物对立、长线伏笔（玉佩/黑影式暗线）、藏锋契约（武神躯/底牌何时显）。
2. 按提问 1 定切割：先写**三幕骨架表**（集区 | 幕/功能 | 季度拍点），标强留存钩集与必要情节锚。
3. 逐集生成，每集一块，标注：**上集承接 / 本集核心事件 / 下集预埋伏笔** + **起承转钩四段**（每段一句剧情 beat）+ **镜号区间**（如 起 S01–S03）+ **时长预算**（每段秒数，合计 90–120s）+ **钩尾类型**（四类择一）。
4. 长线伏笔单列一栏（每条伏笔：埋点集 → 呼应/兑现集），保证跨集呼应不矛盾、藏锋不提前泄底。
5. 全部落 `3_大纲/arc_outline.md`。前 3 集与强钩集写满四段四联；远期集可先一句话方向纲领，逐批 stage-4 regen 解锁详化（仍须标承接/核心/伏笔与钩尾类型）。

---

## 4. 输出物 + 模板

- 落点：`ai_videos/{name}/3_大纲/arc_outline.md`。

### copy-ready 大纲模板（每集一块）

```markdown
### EP{NN} — {一句话集名}
- 上集承接：{接 EP{NN-1} 结尾的什么状态/钩}
- 起（开场钩）：{3s 抓人的开场 beat}  ｜ 镜号 S{aa}–S{bb} ｜ {秒}s
- 承（铺垫）：{信息+情绪累积 beat}     ｜ 镜号 S{cc}–S{dd} ｜ {秒}s
- 转（反转爽点）：{打脸/逆袭/识破 beat}｜ 镜号 S{ee}–S{ff} ｜ {秒}s
- 钩尾（{危机/杀机 | 情感/身世 | 实力/藏锋将显 | 信息/秘密}）：{结尾悬念 beat} ｜ 镜号 S{gg}–S{hh} ｜ {秒}s
- 下集伏笔：{为 EP{NN+1} 预埋的钩/暗线}
- 镜号区间合计：S{aa}–S{hh}（{总镜数}镜）｜ 总时长 {总}s（落 90–120s）
```

### 填好示例（wushen 第二集 · 藏锋版）

```markdown
### EP02 — 当众断绝·暗授武神躯·城门杀机
- 上集承接：接 EP1 觉醒武神系统、硬刚镇北王裴霆与众人后，决意与镇北王府决裂。
- 起（开场钩）：裴知秋当众撂下决裂之言、转身走出厅堂，众人嗤笑其废物放狂言 ｜ 镜号 S01–S03 ｜ 18s
- 承（铺垫）：系统私下暗授【武神躯】（仅主角可见对话框、当集不显化、众人看不出），主角刻意藏锋；凌虚子远眺隐隐生疑却看不透 ｜ 镜号 S04–S09 ｜ 36s
- 转（反转爽点）：离府赶路、系统初引（远离是非·静处淬体·前路机缘），断裂家纹玉佩忽透微温——废物背后另有玄机 ｜ 镜号 S10–S12 ｜ 21s
- 钩尾（危机/杀机）：城门暗处按刀黑影锁定其背影（引 EP3 城门截杀） ｜ 镜号 S13–S14 ｜ 14s
- 下集伏笔：城门黑影=裴昭派的杀手（EP3 截杀）；玉佩微温=身世/传承暗钩（远期呼应）；武神躯藏锋威能留待 EP25"众人才发现"。
- 镜号区间合计：S01–S14（14 镜）｜ 总时长 89s（落 90–120s）
```

> 强钩集（如 EP1/EP3/EP7/EP25/EP30…）须把四段四联写满；远期集可暂留一句话方向纲领 + 钩尾类型，逐批 stage-4 解锁。长线伏笔（藏锋武神躯·石碑"短命"误判·玉佩/黑影暗线）另列「长线钩」栏：每条标埋点集→兑现集。

## 5. QC 关卡（blocker 清零才进阶段 4）

大纲生成后，**强制**跑两道审查（严格度=严）：

1. **`ai_videos__剧情连贯`**（整集层）：每集内 P1–P6 情节链通 / 动机可信 / 爽点铺垫落地 / 系统互动 / 共情 / 设定自洽。
2. **`ai_videos__全剧序列`**（全剧层·超集）：把所有 EP 开场/结尾 beat 与签名台词排成一条线通读，过 R2–R7——
   - 跨集承接顺、无开场重启（后集只承接前集后果、不重演已收尾的戏）；
   - 钩子落点清晰、每集结尾有钩；
   - 爽点/实力单调递进、无倒退、无提前泄底（守藏锋契约：武神躯不在 EP25 前暴露）；
   - 无跨集重复宣告 / 签名台词唯一；
   - 长线伏笔跨集呼应自洽、专名时序对。

发现问题当场 surgical 改（保留更早一处、后处改承接式，顺着往后改受影响集）。**两道审查 blocker 全部清零**方可进阶段 4；否则回修重审。

---

## 6. 避坑

- **强制标镜号区间**：每集每段必带 `镜号 S{aa}–S{bb}`，防阶段 5 分镜时镜与镜衔接断层、镜号错位。
- **每集结尾必预埋钩**：无钩集是 blocker；钩尾必从四类择一并写明引向下一集的什么。前 3 集（72h 高热窗）钩密度最高。
- **长线伏笔跨集呼应不矛盾**：玉佩/黑影/藏锋武神躯/石碑"短命"等暗线须在「长线钩」栏统一登记埋点集→兑现集，逐集核对不断线、不矛盾、不提前泄底。
- 不引入 world.md 未有的设定；不在台词/纲领里提前用尚未登场的专名。
- **完播率节奏律（红果分账=播放×停留×完播）**：每集守「黄金前 3 秒设钩 + 15s 一反转 + 30s 一推进 + 结尾 10s 卡点」，禁冗长铺垫——见 ai_video.md §14.3。
- **每集标 AI 强项 beat（扬长避短）**：每集大纲点明其奇观/打斗/变身/爽点反转 beat 的位置当钩子，避免整集靠纯情感对话支撑（AI 弱项）——见 ai_video.md §14.1。

## 8. 机制（贯穿）

- **每次 update 默认复核**：任何对 `arc_outline.md` 的改动/重生，默认跑受影响范围的本阶段 QC（`剧情连贯` + `全剧序列`，改动牵动相邻集/伏笔时尤甚）+ 在 `specs/ai_video/{name}/changelog.md` 记一条。
- **反馈→进化**：用户每条实战反馈 → surgical 更新本 playbook / 对应审查 skill / `agent_refs`，并记一条"教训"（带来源集号 + 反馈引用）。流程越用越强。
