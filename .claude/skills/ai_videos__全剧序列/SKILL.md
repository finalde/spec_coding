# 全剧序列大师（arc master）

把 `ai_videos/{name}/` **所有 EP 当成一条线通读**，审查并**直接修改**只有跨集视角才能发现的问题。局部相邻镜检查（归 `剧情连贯`）抓不到"整段开场在重复上一集已经收尾的戏"——典型坑：EP1 末已让主角撂下决裂宣告并离场感，EP2 开场又来一段更长的决裂宣告 → 跨集重复宣告、开场像重启。必须**通读整条序列**才能发现。发现问题 → **保留更早（已出片/更靠前）那一处**，把后一处改成承接式，surgical 改后集，不动已定稿前集（除非用户许可重渲染）。

本 skill 是 CLAUDE.md「全剧序列 review」的可执行版 + `ai_video.md` 12.4-D S3（跨集专名时序）。由 `ai_videos__审查总编排` 在全剧层调用（最后一道工序）。

## 何时用
- 用户说"统一 review 所有 EP""跨集重复/重启""开场像把上一集又演一遍""签名台词用滥""爽点没递进/提前泄底"。
- 定稿/重生某集、改任一集开场或结尾时（CLAUDE.md 触发时机）；`ai_videos__审查总编排` 全剧层工序。

## 输入
- 范围：整剧（全部 `episodes/ep*/`）。
- 必读：每集 `script.md` 首尾段 + 首镜/末镜 + `dialogue.md`；`arc_outline.md`（钩与爽点全局布局）；角色 bible（锁定描述符 + voice_id + 身份伏笔）；`world.md`。

## 审查标准（按集号排成序列通读）

| # | 准则 | 不合格信号 | 怎么改 |
|---|---|---|---|
| R1 | **抽取序列** | —（流程项） | 把每集**开场 beat + 结尾 beat**（script 首尾段 + 首镜/末镜台词与剧情）和**全部角色签名台词**按集号排成一条线 |
| R2 | **跨集查重 / 重启** | 相邻集开场在重复上一集已收尾的 beat/已说过的宣告；同一名场面台词或同义宣告两集各来一遍；开场像把上一集结局又演一遍（"你不是已经走了么"） | 前集已收尾的戏（决裂/立誓/觉醒/离场）后集只承接其**后果**、不重演不再宣告；保留更早一处、后一处改承接式（默剧承接/推进式新 beat） |
| R3 | **签名台词唯一** | 标志台词被多镜/多集复用稀释（同一句多个"名场面"出处） | 同一句只留一个出处，其余改写或删 |
| R4 | **爽点/实力单调递进** | 爽点/实力跨集倒退、原地踏步、或提前泄底（藏锋契约被破） | 调整跨集布局，保证递进无倒退、不提前暴露武神躯/底牌（守藏锋契约） |
| R5 | **伏笔/身份呼应** | 身份/伏笔钩跨集矛盾或断线 | 对齐呼应，补或改后集钩子使前后自洽 |
| R6 | **专名时序（跨集 S3）** | 某集台词出现要更后 arc 才登场的人/物专名 | 等其入场再用，提前处换可指代 |
| R7 | **锁定/voice_id 叙事一致** | 同角色跨集形象/声口设定漂移（叙事层面；byte 级不一致归 `格式契约` K7/K8） | 对齐角色 bible 的人设/声口；冲突按 bible 改后集 |

## 工作流
1. 读每集首尾 + 签名台词 + arc_outline + bible，按集号排成序列。
2. 通读过 R2–R7，列跨集不合格项（cite 集/位置/准则）。
3. **当场 surgical 改后集**：重复/重启处保留更早一处、后处改承接式（顺着往后改受影响的镜，空间/状态单调连续）；签名台词去重；爽点递进修正；专名时序修正。
4. **不动已定稿/已出片前集**（除非用户许可重渲染）；三处同步改动集的 shot/dialogue.md/script.md。
5. 复验：无跨集重复/重启、签名台词唯一、爽点单调递进无泄底、伏笔自洽、专名时序对。

## 硬约束
- 本 skill 是局部连贯（`剧情连贯`）的**超集**——多集叙事在宣布"连贯"前必须跑一遍。
- 静音/改默剧不算修好——背后跨集情节/走位不合理要顺着往后改（CLAUDE.md 全剧序列 review 第 5 点）。
- 默认改后集、保前集；要改已定稿前集须用户许可。
- byte 级锁定/voice_id 一致性归 `格式契约`；本 skill 只管叙事层一致。
- 审计写 `.audit/.../events.jsonl` 的 `validation.issue.raised`（level=arc）。

## 输出
- 全剧序列表（每集开场/结尾/签名台词）+ 跨集 patch 清单 + 已落地改动 + 一句话复验结论。
