# 首尾帧 reference — 承接帧 / 首帧·尾帧 prompt 写法（AI 短剧通用）

> 图生视频（Seedance / Kling 全能参考）**跨镜承接帧**与**首帧 / 尾帧锁定**的权威写法。跨项目通用，是 `ai_videos__运镜` M8（承接判定 + 写 `衔接:`/`参考:`/`动作:`/`镜头:`）、`stage5_分镜`（逐对判承接 / 硬切）、`stage6_prompt`（落承接三处 + 结构化首末帧指令）的取词依据；机械校验归 `ai_videos__格式契约` K26 / K27。
> 与 `ai_video.md` 12.4 §承接契约 (A)–(I)+(H2) 同源——本文件是**面向"prompt 怎么写"的展开版 + 正反例**；条款冲突以 `ai_video.md` 为准。

---

## 0. 一句话结论

承接镜的目标是「**下一镜首帧＝上一镜成片末帧**、画面一气接续」。做坏的两大根因：① **首帧没被锁成第 0 帧**（进错槽 / 双帧指令让模型渐变到不存在的末帧）→ 生成起幅漂几十帧；② **seam 落在运动中途**（走路半步 / 转身半圈）→ 模型重置动作相位→跳帧。本文件就是消灭这两类问题。

## 1. 先判定：承接 vs 硬切（默认硬切）

相邻两镜 A→B 满足**全部**三条才做承接，否则硬切：
1. 同场景 / 同 bg plate；
2. 机位连续或渐变（同轴微推 / 拉 / 移、match cut），**非**越轴 / 正反打 / 景别跳切 / 换场；
3. 动作或情绪不间断（B 从 A 末帧那一刻"长"出来）。

换场 / 时间跳 / 回忆进出 / 越轴 / 正反打 = **硬切**（独立首帧、不挂承接帧）。**每集首镜恒为硬切**（无上镜可承）。硬切观众不觉突兀，别为"消顿挫"强行做无缝。

## 2. 三种帧模式 × 结构化指令字段

`参考:` 行用 `本镜首帧=>` / `本镜末帧=>` 作进槽 handle（缩短形，**不写** `(上一镜末帧)`/`(重生时上传)` 这类人读括注）；在 `参考:` **下方另起一行**写结构化指令告诉模型画面起止：

| 镜类型 | 上传 | `参考:` handle | 结构化指令行 |
|---|---|---|---|
| 硬切独立首帧 | 角色 + 场景 | 无 | **不加** |
| 承接镜·**首次生成** | 首帧(=上镜末帧) + 角色 + 场景 | `本镜首帧=>` 置最前 | `首帧: 画面以首帧为起始、自首帧那一刻续起自然顺接展开、起始不重新缩放/不重新定位/不重摆姿` |
| 交接源·**重生锁末帧** | 末帧(本镜 lastframe) + 角色 + 场景 | `本镜末帧=>` 置末 | `末帧: 画面以末帧为结束、收束至末帧` |
| 既承接又尾锁·**重生** | 首帧 + 尾帧 + 角色 + 场景 | 首端 `本镜首帧=>` + 末端 `本镜末帧=>` | `首末帧: 画面以首帧为起始、以末帧为结束、自首帧顺接渐变至末帧` |

## 3. 九条铁律（直接消灭"差几十帧"/接缝卡顿）

1. **首帧进专用「首帧」槽**（image-to-video 首帧图），**不是**全能参考 / 普通参考图槽。进全能参考槽＝模型当"引导图"而非第 0 帧→照样漂。
2. **承接镜＝尾锁源时，首次生成只走「首帧单图」（H2）**。首次生成本镜末帧**根本不存在**；若写「首+末双帧」指令（`…渐变至末帧` + `@图N为尾帧`），等于命令模型渐变到不存在的末帧→首帧不被锁死→起幅漂几十帧。**双帧指令只留给重生**（此时 `shot{NN}_lastframe.png` 已存在作尾帧上传）。`参考分配:` 明写"首次只上传首帧+角色+场景、不传尾帧；尾帧仅重生时上传"。
3. **seam 落点＝静定 beat，禁落运动中途。** 交接帧设计在**收势 / 立定 / 亮相定格 / 反应 beat**（本就该停顿的瞬间），**绝不**停在走路半步（mid-stride）/ 转身半圈（mid-turn）/ 出招中途——i2v 接续半步姿态会**重置步态/动作相位**致跳帧。故**交接源镜的末拍要收到一个静定 beat**（如"转身完成、面南立定收势一拍"），而非停在仍在位移的中途帧。
4. **交接源末段＝锁构图 + 留环境微动，别冻成截图。** 末段把**方向性运动**（行走/推拉/摇移/转身）收止、机位锁死（不位移不变焦）、站位/朝向/姿态/景别不变；但**保留环境级微动**——风掀发梢/衣摆下缘、肩背随一次极缓呼吸轻起伏。纯 dead-still 会像卡住/截图；环境微动无方向性速度、不构成"运动突停"，末帧仍＝B 首帧（可 byte 承接）。
5. **承接镜首拍禁重演已完成的过渡动作（含台词）。** 那个过渡动作（转身/坐下/起身/闭眼/睁眼/抬头/推门/抽剑…）在上一镜**已做完并定格在交接帧上**。本镜 `动作:` 首拍只写**从该已到达状态续起**的下一拍（首帧已立定收势→"自立定起步"，**禁**再写"转身/站定"），并加括注「首帧已是 X 态·自此续起、不重演 X」作模型守卫；`台词:`（尤其 OS）**不复述**刚做完的动作。
6. **首末帧反差铁律。** 运动量＝起始帧→结尾帧视觉差，越接近越静止。每镜首↔末必有显著 delta，杠杆按大小：①景别推/拉 ②位置/姿态 ③表情/面色 ④光线 ⑤机位角度。承接镜首帧被锁（＝上镜末帧），故 delta 由**本镜末帧**与该锁定首帧拉开。
7. **帧源＝截上一镜实际成片末帧，禁另生成静帧。** 另画的"衔接静帧"与实际渲出帧必然漂移、接缝照跳。webapp「⏮ 生成末帧」会把 A 末帧同时写进下一镜 `shot{NN+1}_firstframe.png`，承接交接一键完成。
8. **prompt body 禁写跨镜 / 工作流 meta。** `承 SN` / `接 shotN` / `供下镜承接` / `(重生时上传)` / 日期 / follow-up 编号等一律删——模型只生成单镜、读不懂跨镜引用、纯占字数；正向提"字幕"还会反噬。
9. **≤2000 中文字符**（整个 ```text 块、情节含在内、无例外）。承接相关补字易超顶，去重复述（锁定描述符已在 `角色:` 的不在 `参考分配:` 再抄）。

## 4. 出片端接缝抹平（兜底，治标）

`ai_video_management` webapp 整集 concat 默认按各 shot `衔接:` 字段自动抹平：承接镜用 freezedetect 裁掉头部静止重复帧 + 裁 A 尾部死运动；可选 RIFE 在 A 末段↔B 头段补光流帧（唯一真正修复"定格→重启"运动断裂的手段）。但**接缝根因是设计端**——seam 落静定 beat（§3.3）才是根治，出片端只兜底。越轴/正反打/景别跳切**本就该硬切**、别强行补帧。

## 5. 落点清单（机械可校，归 K26/K27）

- **承接镜三处**：① `## Shot context` 加 `衔接: 承接 shot{NN} 末帧（首帧＝上一镜末帧）`；② `参考:` 行**最前**加 `本镜首帧=>` + 下方结构化指令行；③ `Reference uploads` 加「上一镜末帧（从 shot{NN} 成片截取最后一帧、作本镜首帧上传）」。
- **交接源两处**：① `## Shot context` 加 `尾帧锁定:` 行；② `参考:` 行**末尾**加 `本镜末帧=>` handle。
- 硬切镜：`衔接: 硬切（独立首帧）`，**不得**挂任何承接 / 尾帧 handle。

## 6. 正反例 — ep1 shot10 → shot11

- **✗ 坏**：shot10 末帧＝「已朝南门**前行两三步**」（走路中途）；shot11 写「首+末双帧」指令（`首末帧:…渐变至末帧`+`@图4尾帧`）但首次生成无本镜末帧。→ 首帧不锁、模型自走路半步重置步态，生成起幅相对所给首帧**差 50+ 帧**。
- **✓ 好**：shot10 末帧改＝「转身完成、**面南立定收势站定**」（静定 beat·未迈步·末段锁机位 + 留衣摆/马尾环境微动）；shot11 走「首帧单图」指令（`首帧:…自首帧那一刻续起、起始不重摆姿`）、`参考分配` 首次只传首帧+角色+场景不传尾帧、`动作` 首拍「(首帧＝面南立定收势·自此起步续行、不重演转身) 自立定起步…」。→ seam 落静定、首帧锁死、模型自立定起步不接半步。
