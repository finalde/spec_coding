## 关键契约提醒（写入对应 prompt 文件 annotation）

- **Shot 01**：缩略图契约 FR-29 — t≈2 s 金光迸发峰值帧必须独立可读为合格 9:16 Shorts 缩略图（YouTube Shorts feed 自动抽帧）。两个 prompt 文件（`shot01_kling.md` + `shot01_seedance.md`）顶部均加 `# 缩略图契约` 注释块。
- **Shot 03**：金箍棒钎柄微弱内发光 t=4–6 s 唯一例外（`#F2A65A` rim, ≤5% 表面）；其它镜头金箍棒严格反射环境光、无自身外发光。
- **Shot 05**：回环契约 FR-30 — 末帧构图与 Shot 01 帧 0 byte-identical（相机角度 / 焦距 / 主体位置 / 道具位置全等），仅光照状态翻转（金光迸发 ↔ 金光淡入余烬）。两个 prompt 文件顶部均加 `# 回环契约` 注释块。
- **Shot 04 → Shot 05**：唯一非 hard-cut 转场 — 使用 `匹配剪辑 match cut`（金箍棒姿态 + 镜头垂直升降运动衔接）。
- **运镜 token 锁**：每个 prompt 文件 `镜头:` 字段必须使用 `style_guide.md` § 5 中 5 锁定运镜 token 之一（极缓推近 / 慢速广角揭示 / 轨道环绕 / 手持轻微浮动 / 垂直升降）。

## 与 prompts/ 的关系

本清单是 prompts 编写的首要锚点。每个 `prompts/shotNN_*.md` 必须：

1. `时长:` 字段值与上表「双工具时长映射」一致。
2. `角色:` 字段下 byte-identical 粘贴 `characters/main.md` 锁定描述符块。
3. `场景:` 与 `光线/色调:` 字段使用 `style_guide.md` 调色盘 hex + 光线 token 词汇。
4. `镜头:` 字段使用上表与 style_guide § 5 锁定运镜 token。
5. Kling 文件包含 `negative_prompt:` + `[参考图: ...]`；Seedance 文件包含 `约束:` + 时间线分段动作（不得包含 `negative_prompt:` 或 `[参考图: ...]`）。
