# Follow-up draft 016 — 2026-06-13

Add a **Prompt Lab** section to the `spec_driven` webapp surfacing the repo-root `prompt_lab/` library (category subfolders, each holding independent task `.md` files), with overview + simple CRUD + prompt highlighting and source/expected-result links.

## Abstracted instruction

The repo root has a `prompt_lab/` folder: a personal library of copy-paste, autonomous "Ralph-loop" AI build prompts, organized as `prompt_lab/{category}/{task}.md`. Surface it as a first-class section of the `spec_driven` webapp.

Requirements:

1. **New "Prompt Lab" section / page.** Reachable from the landing page (a Prompt Lab entry alongside the project picker) at route `/prompt-lab`. It does NOT depend on an active spec project — `prompt_lab/` is its own top-level surface.

2. **Overview of all md info.** The page lists every `prompt_lab/**/*.md`, grouped by category, as cards. Each card shows the parsed metadata: title, the `Stack / Est / Output` line, the **Source** link, the **Expected result** link/image, and a preview of the copy-paste prompt. Clicking a card opens the file.

3. **Simple CRUD.**
   - Create: a "New prompt" action that creates a new `.md` under a chosen (existing or new) category.
   - Read: open a file in the existing reader.
   - Update: edit + save through the existing file editor.
   - Delete: a delete action per md (with confirmation).

4. **Prompt highlighted in read mode, click to edit.** When viewing a `prompt_lab/` md, the copy-paste prompt code block renders **highlighted** (visually emphasized as the primary artifact) and is **clickable** — clicking it switches the file into edit mode.

5. **Source + expected-result links (md AND UI).** Every task `.md` carries a parseable `**Source:**` link (original inspiration) and an `**Expected result:**` link/image (a representative reference of what the prompt should produce). Both are surfaced in the Prompt Lab UI.

## Backend shape

- Extend the path sandbox (`safe_resolve` allowed top-level + `exposed_tree.is_inside`) to admit `prompt_lab/` so the existing `GET/PUT /api/file` serve read + update + create-in-existing-category for `prompt_lab/*.md`.
- `GET /api/prompt-lab` — overview: categories → entries with parsed `{path, name, title, meta, source, expected, prompt}`.
- `POST /api/prompt-lab/file` — create a new md (ensures the category dir; refuses overwrite).
- `DELETE /api/prompt-lab/file` — delete a single md, scoped to `prompt_lab/`.
- The generic `DELETE /api/file` stays 405; deletion of `specs/`/`projects/` files is NOT opened up. Deletion is `prompt_lab/`-scoped only.

- `prompt_lab/` is a personal scratch library outside the spec-driven `specs/`/`projects/`/`ai_videos/` surfaces; this follow-up gives the webapp read/write access to it as a new sandbox root, parallel to (not under) the Specs section.
- Routes are added to the existing single `apps/api/routes.py` to match the current code's organization (the per-aggregate `routes/` split from follow-ups 014/015 is a future stage-6 regen, not yet applied to the live code).
