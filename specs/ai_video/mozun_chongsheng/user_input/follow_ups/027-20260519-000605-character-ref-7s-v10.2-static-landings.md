## Why

User report this turn after rendering v10 character mp4s + clicking 🖼 extract:
> "the side is still almost front, the back picture actually shows side ... the video does not have a backview in it, I think it start to move around 4~5s so the last frame in the video is still side view"

Diagnosis: video model under-rotates v10's single 4s continuous orbit. Spec was 45°/s × 4s = 180°; empirical was ~22°/s with motion-start delay to ~4-5s, so the 7s video never reaches 180° back. Root cause is structural: video models don't honor timed-beat speed instructions for "slow continuous motion" — they interpret slow relative to internal pacing.

v10.2 fix: replace v10's single 4s continuous orbit with **3 static landings + 2 short motion bridges**. Each angle pick lands at a guaranteed-static moment. Angle contract changes from "time × speed" to "explicit landing angle hold".
