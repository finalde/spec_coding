# Follow-up draft 001 — 2026-05-09

Summary: 现有 Seedream 立绘 prompt 输出的图像偏 国漫 / 插画 / 工笔 风格，用户要的是真人 / 写实 / 电影级 风格。全局把渲染样式锁定为 **影视级真人写实**，并把所有 prompt 的 negative 强制禁止 anime / cartoon / illustration / chibi。

## 用户原话

> 你给的prompt生成的人物图片变成漫画了，我要真人的

## 根因分析

stage 6 U1 写 10 份 Seedream 立绘 prompt 时，"风格" 行使用了：
> 传统中国仙侠插画 + 国漫风格 + Character design + 高细节服化道，水墨写意 + 工笔精绣

这些关键词（"插画" / "国漫" / "工笔" / "水墨写意"）让 Seedream 直接输出 2D 漫画 / 国漫风格 PNG。由于 Kling 2.1 Pro 的 image-to-video 是以这些 PNG 作为参考帧 (`input_image_urls`)，Kling 视频也会继承"漫画感"。即使 Seedance text-to-video 没显式说"插画"，但在没有"真人写实"正向锚定的情况下，Seedance 模型同样可能漂移到 stylized output（特别是当主词 "魔尊 / 仙侠" 在训练数据中常见动画作品语境）。

## 期望行为

1. **Seedream 立绘** PNG 输出：影视级真人 / 写实 / 仙侠古装剧造型 / 8K 写实人像摄影 / cinematic — 类似《琉璃》《长月烬明》《苍兰诀》这种电视剧的剧照质感（这些剧本身就是真人剧）。
2. **Kling 视频输出**: 因为 Kling image-to-video 锚定立绘 PNG，所以立绘改了视频自动跟改。但仍要在每镜 Kling prompt 的 `光线/色调` 行加 "影视级真人写实" 正向锚 + 在 `避免` / 负向区禁止 anime / cartoon / illustration。
3. **Seedance 视频输出**: 直接驱动视觉，必须每镜显式加 "影视级真人写实 + cinematic + 4K HDR" 正向锚 + "避免" 区禁止 anime / cartoon / illustration / 国漫 / 插画 / 工笔 / 水墨写意。
4. **现有的 1 处反向写实 bug**: ep01 / shot01 seedance 当前 `避免:` 段含 "**写实毛孔皮肤瑕疵**" — 这是写错了（把"写实"当作要避免的东西），删除该项。
