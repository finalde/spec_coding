## Why

- The 4s turntable gives each character ≈ 38% more on-screen seconds for identity capture (face / 360° silhouette / voice timbre) than the previous 2.9s ceiling.
- Critically: the ai_video_management chars-reel concat (`ShotConcatBuilder._ffmpeg_concat`) trims each character clip to the **first 2 seconds** before concatenation. The new schema front-loads identity by having the character finish saying "一" and "二" within 0-2s — so the 2-second slice the concat operator pulls is still self-sufficient. "三" + face推近 close-up live in 2-4s.
- Switch from "1, 2, 3" (Arabic numerals) to "一, 二, 三" (Chinese numerals). All-Chinese content rule for `ai_videos/{name}/` (CLAUDE.md § AI video rules) preferred + cleaner唇形对齐 in Chinese mouth-shape.

### New 动作 / 台词 contract (replace existing code-block segments)

```
动作（timed beats，4s — 信息密度优先；本视频是 reference，不是给观众看的）:
  - 0-1s: 正面**全身远景**起手，自然站姿，眼神看镜，自然呼吸；说"一"。
  - 1-2s: 镜头**快速 360° 顺时针环绕一圈**（侧面 90° → 正背 180° → 另一侧 270° → 回正 360°），全身始终在画面内，覆盖正/侧/背/侧四向轮廓；说"二"。
  - 2-3s: 镜头由全身远景**推**至**面部中近景特写**（眉眼 + 服装领口 + 标志特征点 row #11）；说"三"。
  - 3-4s: 面部中近景定格 1s（眼神跟镜，自然呼吸，无台词），用于身份特征 final lock。

台词 / 字幕: 内嵌唇形对齐音频（中文 3 个数字 "一, 二, 三"，按动作 beats 顺序播报；音画同步；字幕方正粗黑 白底黑边）。**前 2s 必须包含 "一" + "二" 的完整发音**（下游 ai_video_management 短角色合辑 / ✂ 截到 2s 工具均取前 2 秒）。
  1. "一" （0-1s 起声 / 中性音域 — 声线 timbre 锚点）
  2. "二" （1-2s 中段 / 节奏校准 — 必须在 2s 前结束）
  3. "三" （2-3s 落声 / 咬字校准）
  (3-4s 无台词，仅面部特写定格)
```
