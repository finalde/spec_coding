# Follow-up draft 029 — 2026-05-21

每个 shot 在「Reference placeholders」与「视频 prompt」之间新增 `## 小说文本 / Novel prose` 段：把本 shot 写成 200-400 字的仙侠小说式散文叙述，让 user 顺读全集 11 个 shot 的此段时有"在读一本小说"的连贯体验，而非翻 prompt 摘要。

## 内容来源

仅派生自同 shot 内已有信息（Shot context Summary / `动作:` timed beats / `台词:` / `光线 / 色调:` / `场景:`），不引入 timed-beats 之外的新动作或新角色。

## 笔法要求

- 散文一段（信息密度极大可拆 2-3 段，上限 600 字）。
- 加入感官（魔气腥涩、雷光灼意、长袍沉甸等）、心理留白、节奏短句。
- 台词以「」/ `""` 直接嵌入散文，不再另加角色名 / 字体注解。
- 句首避免重复 "本 shot..." / "镜头中..."，多用情节衔接词（"忽而" / "与此同时" / "话音未落"）。

## @-ref header（首行强制）

第一行必含 @-ref header，列出本 shot 涉及的所有人物与场景，格式：

`{人物1}請參考:@{小说中文名}_{人物中文名}，{人物2}請參考:@{小说中文名}_{人物中文名} ... {场景}請參考:@{小说中文名}_{场景中文名}`

具体到本项目：

- **小说中文名 = `魔尊归来`**（取自 `ai_videos/mozun_chongsheng/README.md` H1 中文剧名；**不是** task_name pinyin slug）。
- **人物/场景中文名** = `## Reference placeholders` 段 placeholder 的中文名部分，**去掉 `cN_` / `sN_` 前缀**。例：placeholder `{ref_c1_沧冥}` → @-ref 中写 `沧冥`；`{ref_s7_山道平台}` → `山道平台`。

例：`沧冥請參考:@魔尊归来_沧冥，长阶顶請參考:@魔尊归来_长阶顶`

人物在前，场景在后；header 后空一行再写散文正文。

## 硬约束（违反 = stage-6 blocker）

- 不得复制 `0-2s: ...` 行；不得出现 `{ref_xxx}` placeholder；不得出现 hex 色号 / fps / 比例 / 镜头景别等技术语；不得用列表 / 表格 / 代码块。
