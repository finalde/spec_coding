# Follow-up draft 022 — 2026-06-02
shot5 生成视频里出现一众大臣, 本不应在镜头里。不知 prompt 哪里有问题让 Kling 加了一堆群众。

## 根因诊断
非 prompt 写错, 是**语义联想致幻**: shot5 `情节`/`台词` 含帝王圣旨措辞「今解除朕与其之婚约。钦此」(`朕`+`钦此`) + `走位` 的「跪礼区」+ 太监宣旨 → Kling 训练数据里「太监宣旨/接旨」场景几乎都是一堂跪伏群臣百官, 于是把空厅自动填满群众。prompt 虽 `情节` 写「空旷的正厅」但没**显式定员**、`负向` 也没禁群众 → 模型按惯例补人。属整个 ep01 接旨戏的通病, 非 shot5 独有。

## 落地 (全 28 shot)
- `走位:` 末尾加正向定员: 「厅内系国公府私宅正厅 (非朝堂金殿), 仅本镜入画人物, 别无他人 —— 无群臣 / 无侍从 / 无围观人群, 不得凭空增添群众。」
- `负向:` 加 `不要 群臣 / 大臣 / 百官 / 群众 / 围观人群 / 跪伏群臣 / 侍从随从 / 多余人物 / 凭空增添人物`。
- 帝王措辞 (朕/钦此) 是剧情台词必需, 不删; 靠定员 + 负向反制人群联想。

## 抽象规则 (common → ai_video.md)
宣旨/接旨/朝会/升堂/婚宴/法事/战阵 等隐含人群的场景, 视频/图像模型会凭空补群众; 须 走位定员 (仅本镜人物/别无他人 + 点明场所非朝堂) + 负向禁群众。
