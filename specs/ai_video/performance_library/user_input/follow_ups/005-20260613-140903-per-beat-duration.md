## 抽象指令

检验视频时长不再全库固定（原 5s），改为**按 beat 需求 per-entry 分配**：短 beat（微表情 / 单拍）保持短，多拍渐进 / 高强度条放长。15s 是上限不是目标——给短 beat 配长时长只会诱发模型编造填充动作（AI 视频长时长的主要失败模式）。

## 实现（已落地）

时长阶梯（`tools/set_perf_durations.py`）：
- 强度1 微表情=4s / 强度2–3 单拍=5s / 强度4 濒临=7s / 强度5 失控=9s。
- 显式多拍渐进条（锁定块有 `[a–b s]` 时间窗）= clamp(round(2.5×拍数), 8, 15)；**时间窗与「总和 N 秒」按比例放大**，使每拍铺满新时长、不留填充空档。
- 全库 38 条：4s×3 / 5s×22 / 7s×7 / 8s×1 / 9s×2 / 10s×2 / 12s×1。perf_0035（5 拍）→12s 时间窗等比放大为 `[0–2.4s]…[10.1–12s]`。
