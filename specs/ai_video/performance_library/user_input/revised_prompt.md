# Revised prompt — performance_library

## 目标

在 `ai_videos/` 下建立一个新的**共享表演演技库**（working name: `_performances/`），与既有的 `_actors/`（演员试镜库）、`_voices/`（声线库）同属下划线前缀共享资产库模式。库的内容是：把人类表演方法提炼成一套 **generic、可复用、与具体剧目无关**的表演 prompt 条目，经过检验视频验证后，供所有短剧项目的 shot prompt 直接 reference。

## 背景与动机

- 现有体系中，角色外观（character bible 一句话锁定 + Seedream 立绘）、场景（scenes/ 场景档）、声线（_voices/）都已资产化、可逐字复用；唯独"表演/演技"维度每个 shot 都是现场即兴撰写，质量不稳定、经验不沉淀。
- AI 视频模型（Kling / Seedance 等）对表演描述的服从度是所有维度中最弱、最不可预测的——因此**经过实测验证**的表演 prompt 才有资产价值，未验证的描述词网上随处可得。

## 期望成果

1. **分类体系**：表演条目按多维度归类（用户认可的方向：情绪 × 强度 × 风格 × 载体——例如 悲伤 × 微表情→失控嚎啕 × 内敛/外放 × 面部/肢体/呼吸/视线），而非"每种情绪几十条"的扁平列表。每种主要情绪下覆盖数十种表演方法。
2. **条目产出物**：每条 entry 的核心是一段**设计为可直接嵌入 shot prompt 字段（`动作:` / `表情:` 等）的锁定文本块**，而非散文式演技分析。
3. **检验视频闭环**：每条 entry 配套检验视频的生成 prompt（控制变量：固定测试演员（取自 `_actors/`）、固定场景、固定机位、固定时长，唯一变量是表演描述），用户渲染后反馈，条目带实测笔记（哪个模型表现好、哪些词被忽略、验证状态），不断迭代修正直至验证通过。
4. **引用机制**：库完成后，任何短剧的 shot prompt 想要某种演技效果时直接 reference 对应 entry（具体引用语法对齐现有 reference-handle 约定，在后续阶段确定）。

## 约束（继承自现有 harness 约定）

- 路径英文/拼音，文件内容中文。
- 库为跨项目共享资产，不属于任何单一剧目；其建设本身作为一个 `task_type=ai_video` 任务走六阶段流程。
- 比例、模板字段等对齐 `.claude/agent_refs/project/ai_video.md` 既有规则；表演库自身的新规则在落地后回写 refs。

## 待澄清（进入 Stage 2 interview）

- 维度集合与取值范围的最终确定；首批覆盖哪些情绪、目标条目数量。
- 库的目录布局与命名（`_performances/` vs 其他名字；entry 编号方案）。
- 检验视频的验收标准（什么算"验证通过"）。
- 引用语法的具体形式；是否进入 12.4 模板成为正式字段。
- 与 webapp（ai_video_management）的集成范围是否在本期。
