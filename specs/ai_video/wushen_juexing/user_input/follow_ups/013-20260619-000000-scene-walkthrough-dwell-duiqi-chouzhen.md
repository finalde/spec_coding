# Follow-up draft 013 — 2026-06-19
核对 scene walk-through prompt 的「逐秒方位」是否与导出截帧时点对得上——发现悬停窗口未覆盖固定截帧时点，已修。

## 指令（抽象后）

- 用户质疑：生成 scene 视频的 prompt 里「几秒对应哪个方向」对不对得上。
- 实查：prompt 确实带逐秒方位（`动作:` 块 + `>` 注），但**悬停窗口与固定截帧时点 1.5/4.5/7.5/10.5/13.5s 没对齐**——北(1.5)/中景(10.5)/东(13.5) 三个截帧点落在运动段，会抽到模糊/偏角帧。
- 修法：把 5 个悬停窗口居中/覆盖到固定截帧时点（首尾 dwell 2s、中间各 1s，dwell 间约 2s 过渡），不加行数、保持 prompt ≤2000 字。`步骤二` 的 `>` 注 + `动作:` 块 + 「背景图系统 index」表「视频秒段」列三处同步。

## 通用规则候选

> scene walk-through 的悬停窗口必须**覆盖固定截帧时点 1.5/4.5/7.5/10.5/13.5s**（抽帧才落在静止段）——这是「截帧时点固定」契约的隐含前提，作者排 dwell 时不能让截帧点落进运动/过渡段。可考虑回填 ai_video.md rule 12.10。（本次先就 wushen 修，规则回填待确认。）
